OpenClaw 2026.4.15-beta.1: 認証状態カードと cloud memory を強化

Model Auth 状態カード、cloud storage 対応の memory-lancedb、Copilot 埋め込み検索をまとめて追加した beta.1 です。

リリース日 2026-04-15
バージョン v2026.4.15-beta.1
ツール OpenClaw

主な変更点

OpenClaw の 2026.4.15-beta.1 は、運用面の見通しと記憶基盤の拡張を同時に進めた更新です。コントロール UI の認証状態表示、クラウド保存に対応したメモリ、Copilot 向けの埋め込み検索が中心になっています。

  • Model Auth 状態カード: Control UI / Overview に OAuth トークンの健全性やレート制限の圧力を見える化するカードを追加しました。
  • cloud memory: `memory-lancedb` が cloud storage に対応し、ローカルディスクだけに依存しない永続メモリ運用がしやすくなりました。
  • 検索とモデル実験: GitHub Copilot の embedding provider を memory search に追加し、`agents.defaults.experimental.localModelLean: true` で軽量ローカルモデル向けの既定ツールを減らせるようになりました。

影響や使いどころ

認証状態の可視化は、運用中の失敗要因を早めに見つけるのに役立ちます。クラウド保存対応のメモリは、複数マシンや長期稼働のワークフローで扱いやすく、localModelLean は弱めのローカルモデルで余計なコンテキストを減らしたいときに効きます。

注意点

今回は beta.1 なので、機能を広げつつも回帰確認は必要です。リリースノートには、機密情報のマスキング強化、設定ハッシュの再読込、古い dist チャンクの掃除などの修正も含まれています。

まとめ

OpenClaw は、エージェント運用の可視化と記憶基盤を少しずつ実用寄りに寄せています。既存の自動化を回しているなら、今回の beta.1 はチェックしておく価値があります。
