OpenClaw 2026.4.10: Codex provider と Active Memory を追加

Codex 管理の認証・実行経路、会話前に関連記憶を取り込む Active Memory、exec policy 管理などが入った大型更新です。

リリース日 2026-04-11
バージョン v2026.4.10
ツール OpenClaw

主な変更点

GitHub Releases では、2026-04-11 付で OpenClaw 2026.4.10 が Latest として公開されています。今回の中心は、codex/gpt-* 系モデルを Codex 管理の認証、native threads、model discovery、compaction で扱う bundled Codex provider の追加です。

影響や使いどころ

Active Memory plugin も大きな追加です。メイン応答の直前に専用 memory sub-agent を挟み、ユーザーの好み、過去文脈、関連する詳細を自動で取り込めるようにする設計です。毎回「覚えて」「記憶を検索して」と明示しなくても、長期会話の文脈を使いやすくする方向の更新です。

補足

CLI では openclaw exec-policy が追加され、ローカルの exec 承認ファイルと tools.exec.* 設定の同期を管理できます。Microsoft Teams、Matrix、Telegram、Feishu、Gateway、browser security、OpenAI/Codex まわりの修正も多く、単一機能ではなく運用基盤全体の強化と見た方がよさそうです。

まとめ

2026.4.10 は、OpenClaw を Codex や長期記憶、複数 gateway 連携の実運用へ寄せる大型リリースです。特に Codex provider と Active Memory は、AI エージェントを継続的な作業環境として使う場合に確認しておきたい変更です。
