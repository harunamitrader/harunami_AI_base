OpenClaw 2026.4.2: Task Flow 管理と origin 検証を強化

Task Flow の永続管理を戻しつつ、WebSocket の origin 検証や provider ルーティングを固めた、運用寄りの大型メンテナンス更新です。

リリース日 2026-04-02
バージョン 2026.4.2
ツール OpenClaw

主な変更点

公式 GitHub Releases の 2026.4.2 では、Task Flow の managed / mirrored 同期、永続 state / revision 管理、openclaw flows による確認と復旧など、背景オーケストレーションを扱う基盤が大きく拡張されました。child task の managed spawning や cancel intent の扱いも入り、長く走るフローをより明示的に管理できます。

あわせて Android assistant 起動、plugin 向け before_agent_reply フック、各種 provider / transport まわりの共通化も追加されています。機能追加だけでなく、設定の持ち方や実行経路を整理するリリースです。

影響や使いどころ

Task Flow 系の更新は、複数タスクを並行で回す運用や、途中停止と復旧を含むジョブ管理で効きます。plugin 経由で管理したいケースでも、runtime 側の Task Flow 境界が明示されたことで実装しやすくなりました。

Fixes では、browser-originated 接続の origin 検証を proxy 条件に依らず強制する変更、OpenAI / Anthropic / Copilot 系の native-vs-proxy 判定の一元化、loopback 実行まわりの失敗修正が含まれます。Gateway や provider を自前設定している構成ほど影響確認が必要です。

注意点

Breaking changes として、xAI の x_search 設定と Firecrawl の web_fetch 設定パスが plugin-owned の新しい場所へ移りました。既存設定を持つ環境では、リリースノート記載の openclaw doctor --fix を前提に差分確認したほうが安全です。

まとめ

OpenClaw 2026.4.2 は、Task Flow を本格運用しやすくしつつ、origin 検証と provider ルーティングを締め直した更新です。見た目よりも内部運用に効く変更が多く、OpenClaw を継続利用しているなら優先して確認したいリリースです。
