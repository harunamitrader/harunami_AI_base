Slack から Copilot を使って Issue を作成・管理可能に

GitHub は 2026年3月30日、Slack 連携における GitHub Copilot の大幅なアップデートを発表しました。これにより、Slack 上のチャットから直接、自然言語で GitHub Issue を作成・ブラッシュアップできるようになります。

主な変更点

  • 自然言語による Issue 作成: @GitHub にメンションを送り、やりたいことを自然言語で伝えるだけで、構造化された Issue 案が生成されます。
  • サブIssue（子課題）の同時生成: 複雑なタスクを依頼した場合、自動的に親課題とそれに紐づくサブ課題を切り分けて作成できます。
  • スレッド形式の対話型作成: Slack のスレッド内で Copilot とやり取りしながら、Issue の具体化や詳細の修正が可能です。
  • Slack 内での詳細表示（フレックスペイン）: 作成した Issue の詳細を、ブラウザを開くことなく専用パネルで確認・編集できます。
  • チャンネルごとのデフォルト設定: チャンネルごとにデフォルトの送信先リポジトリを指定可能になり、チームごとのワークフローをスムーズにします。

まとめ

今回のアップデートにより、開発チームは Slack での議論からシームレスにタスク管理（GitHub Issues）へ移行できるようになります。コンテキストを維持したまま、Copilot が適切なドキュメント化を支援することで、開発効率のさらなる向上が期待されます。
