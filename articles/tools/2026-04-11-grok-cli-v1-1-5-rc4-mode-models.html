Grok CLI v1.1.5-rc4: modeModels でモード別モデルを設定可能に

ユーザー設定の modeModels により、用途ごとの実行モードに対して既定モデルを割り当てられるようになったリリースです。

リリース日 2026-04-11
バージョン v1.1.5-rc4
ツール Grok CLI

主な変更点

公式 GitHub Releases では、2026-04-11 付で grok-dev@1.1.5-rc4 が公開されています。変更点は、ユーザー設定の modeModels によって per-mode default models を指定できるようになったことです。

影響や使いどころ

コーディング、計画、調査、軽量な質問など、モードごとに必要なモデルの強さや速度は変わります。modeModels を使うと、毎回モデルを手で切り替える代わりに、作業モードごとの既定値を設定しやすくなります。

補足

rc3 では LSP support と diagnostics が入っており、rc4 はその上に設定粒度を足す小さめのリリースです。まだ rc 系なので、本番ワークフローでは既存設定との互換性を確認してから導入するのが安全です。

まとめ

v1.1.5-rc4 は、Grok CLI を用途別に使い分けるための設定改善です。複数モデルを使い分けるユーザーほど、実行モードとモデル選択の対応を固定できるメリットがあります。
