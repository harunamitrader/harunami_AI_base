Grok CLI v1.1.5-rc5: MCP stdio stderr を TUI から分離

MCP stdio の stderr を TUI から分離し、offline mode 検出や permissions 表示も整えた安定化リリースです。

リリース日 2026-04-13
バージョン v1.1.5-rc5
ツール Grok CLI

主な変更点

Grok CLI の公式 GitHub Releases では、2026-04-13 付で v1.1.5-rc5 が公開されています。大きなポイントは、MCP stdio の stderr を TUI から切り離し、通信ログやエラー出力が画面表示を乱しにくくなった点です。

影響や使いどころ

MCP server を複数つないで CLI を運用している環境では、stderr の扱いが安定すると、エージェント画面と外部ツールの診断情報を分けて追いやすくなります。TUI を常用する場合の見通しが上がる更新です。

補足

offline mode の検出も改善され、ネットワーク状態に応じた挙動を把握しやすくなっています。さらに permissions 表示や git status 表示まわりの調整も含まれており、rc 系列として実運用前の細かな安定化が続いています。

まとめ

v1.1.5-rc5 は、目立つ新機能よりも MCP / TUI / offline detection 周辺の実用性を上げるリリースです。MCP 連携を前提に Grok CLI を使っている場合は確認しておきたい更新です。
