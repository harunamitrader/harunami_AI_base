RuView

WiFi 信号の乱れを解析し、カメラなしで壁越しに人間の姿勢推定やバイタル測定を行う Rust 製システム。

記事作成日 2026-03-26
作成時順位 4位
対象 repo ruvnet/RuView

これは何か

RuView は、WiFi 信号（CSI: Channel State Information）の干渉を解析し、視覚（カメラ）を一切使わずに人間の位置や姿勢、心拍数、呼吸数などを特定する高精度なセンシングシステムです。Rust 言語で書かれており、従来の手法を遥かに凌ぐ実行速度を実現しています。

何ができるか

電波が障害物を透過する性質を利用し、壁の向こう側にいる人の心拍数（40-120 BPM）や呼吸（6-30 BPM）をリアルタイムで測定できます。さらに WiFi DensePose 技術により、人の 3D ポーズ（姿勢）まで推定可能です。カメラ画像を使用しないため、夜間や視界の悪い環境でも動作し、何よりプライバシーを侵害しないという大きなメリットがあります。

目立つポイント

Rust による実装により、システム全体の遅延が 50ms 未満と極めて低く、元の Python 実装と比較して 800 倍という驚異的なサンプルレートを達成しています。また、教師あり学習（カメラ画像）なしで WiFi データから直接自己学習を行うアーキテクチャを採用しており、設置環境に柔軟に適応できます。

セットアップや使い方の流れ

ESP32 などの安価な WiFi マイコンをセンサーノードとし、収集されたデータを RuView の解析サーバー（Rust 実行環境）へ送信します。複数のセンサーを「メッシュ」状に配置することで、ブラインドスポットを消し、より正確な空間把握が可能になります。

どんな人向けか

介護施設でのプライバシーに配慮した見守りシステム、スマートホームにおける在室検知、あるいは災害時のガレキ下に閉じ込められた生存者の探索（WiFi-Mat 拡張）など、人命救助やヘルスケアの分野で革命的な価値を提供します。

注意点

WiFi 信号の反射や干渉を利用するため、金属製の家具が多い環境や、極端に複雑な多重反射が起こる場所では精度が低下する恐れがあります。また、CSI データの取得には対応したチップセット（Intel NIC や ESP32 等）が必要です。

まとめ

WiFi を単なる「通信の道具」から「視覚の代替」へと塗り替える RuView。Rust の高いパフォーマンスと革新的な電波解析 AI を組み合わせた、次世代のアンビエント・コンピューティング基盤と言えます。
