これは何か

Production SaaS Starter は、中〜大規模な B2B SaaS を迅速かつ堅牢に立ち上げるために設計された、本番導入レベルのプロジェクトテンプレートです。バックエンドに Go、フロントエンドに Next.js を採用し、アーキテクチャには保守性の高さで知られる Hexagonal Architecture を全面的に導入しています。単なる「動くもの」ではなく、10年後もメンテナンス可能な構造を目指した、エンジニア志向の強いスターターキットです。

何ができるか

  • B2B 向けマルチテナンシー： 企業（テナント）ごとのデータ分離や、組織管理機能を標準で備えています。
  • モジュラーモノリス設計： 各機能をモジュールとして独立させつつモノリスとして管理。マイクロサービス化への移行パスを確保しつつ、開発の摩擦を最小化します。
  • 堅牢な実装パターン： DTO（Data Transfer Object）や Repository パターンの徹底により、ビジネスロジックと外部インフラ（DBなど）を明確に分離した「疎結合」な実装が学べます。
  • 組織的な権限管理（RBAC）： 管理者、メンバーなどのロールベースのアクセス制御を、Go ベースのバックエンドで型安全に実装済み。
  • モダンな開発環境： Docker Compose による環境構築、Taskfile による共通タスクの自動化、データベースマイグレーション管理などが初めから統合されています。

目立つポイント

世の中には多くの SaaS テンプレートが存在しますが、その多くは「機能を盛り込む」ことに注力しています。一方このプロジェクトは、「正しい設計を維持する」ことに重きを置いています。特に、依存関係の逆転を活かしたヘキサゴナルアーキテクチャの具体的な Go 実装例は、大規模開発に携わるエンジニアにとって非常に貴重な教材となります。

セットアップや使い方の流れ

  1. リポジトリをフォークまたはクローンします。
  2. docker-compose up でデータベース等のインフラを起動します。
  3. Taskfile (task コマンド) を使用して、バックエンドとフロントエンドの依存関係を解決・起動します。
  4. 同梱されているリファレンス実装（User 管理や Auth 周り）を参考に、独自のビジネスロジックを追加していきます。

どんな人向けか

  • スタートアップの CTO やリードエンジニアで、将来の拡張を見据えた最適なアーキテクチャで SaaS を始めたい人。
  • Go を使った「正しい大規模設計」のベストプラクティスを、動くコードを通じて学びたい開発者。
  • 認証、組織管理、RBAC などの「車輪の再発明」を避け、ビジネスの本質となる機能開発に集中したい人。

注意点

高度な抽象化が行われているため、小規模な使い捨てアプリを開発する場合にはオーバーエンジニアリングに感じる可能性があります。アーキテクチャ自体が学習コストを要するため、チームメンバー全員がヘキサゴナルアーキテクチャの意図を理解して運用することが、本プロジェクトのメリットを最大化する鍵となります。

まとめ

Production SaaS Starter は、エンジニアによるエンジニアのための SaaS 基盤です。技術選定からフォルダ構成、通信プロトコルに至るまで「本番で通用するか」という基準で削ぎ落とされたその構造は、次世代の Web システムを構築する上での強力な武器になるはずです。
