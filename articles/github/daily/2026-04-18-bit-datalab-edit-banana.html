Edit-Banana

「編集できない画像」を「編集できる図表」へ。SAM 3 とマルチモーダル LLM を駆使して、静的なチャートを DrawIO や PPTX に高精度復元するフレームワーク。

これは何か

画像形式（PNG/JPG等）で保存されてしまったフローチャート、グラフ、構成図などを、構造解析によって「再度編集可能な形式（DrawIO XML, PowerPoint, LaTeX等）」へと書き戻すユニバーサル・コンテンツ・再エディターです。Meta の最新セグメンテーションモデル SAM 3 をファインチューニングして使用し、図の構成要素を個別に分離・認識することで、レイアウトや論理関係を維持したまま、1:1 の高精度なデジタルツイン（編集可能データ）を生成します。

何ができるか

  • リ・エディタ機能： 過去の資料から画像だけが残っているフローチャートなどをアップロードすると、個々のボックスや矢印が独立したオブジェクトとして扱える DrawIO 形式で出力されます。
  • 高度な要素分離 (SAM 3)： 複雑に重なり合った図形や多種多様な矢印のスタイル（点線、太さ、色）を正確に識別し、ベクターデータとして再構成します。
  • マルチモーダル LLM スキャン： マルチモーダル大規模言語モデル（VLM）を用いて図全体を複数回スキャンし、図の中に含まれる情報の論理的な階層や接続関係を補完します。
  • 強力なテキスト & 数式認識： 単なるテキスト OCR だけでなく、Pix2Text を統合することで複雑な数学公式も認識し、LaTeX 形式で埋め込むことが可能です。
  • ビジネス資料への転載： PowerPoint (pptx) への書き出しにも対応しており、画像しか手元にない古いグラフを、最新の数値を反映させるための編集可能なグラフへと即座に変換できます。

目立つポイント

「画像の中の文字を抽出する」ツールは多くありますが、Edit-Banana は「図の中の構造（意味）を抽出して、元の編集ソフトで開ける形式に戻す」という一歩踏み込んだ解決策を提示しています。特に SAM 3 を図表のセグメンテーションに特化させて利用している点は技術的にも非常に興味深く、業務効率化の観点でも「資料の作り直し」という苦痛から解放してくれる救世主的なツールです。

セットアップや使い方の流れ

  1. リポジトリをクローンし、requirements.txt をインストールします。CUDA 対応 GPU が推奨されています。
  2. SAM 3 のモデルウェイト（sam3.pt）を指定の場所にダウンロードし、環境をセットアップします。
  3. コマンドラインから python main.py -i <画像パス> を実行します。
  4. output/ ディレクトリに、生成された DrawIO XML や中間ファイルが出力されます。
  5. オンラインデモ（editbanana.anxin6.cn）も公開されており、ブラウザ上ですぐに変換を試すことも可能です。

どんな人向けか

  • 画像しか残っていない過去のシステム構成図やフローチャートを、最新の状態に修正・メンテナンスしたいエンジニアやドキュメント作成者。
  • 論文やレポート内の高度なグラフを、自分のプレゼン資料用に微修正して引用したい研究者。
  • 「画像化された図表」を構造化データとして抽出して機械学習などの学習データに活用したいデータサイエンティスト。

注意点

非常に高度な処理を行うため、ローカルで動かすには CUDA 環境（GPU）が必要です。また、GitHub 版はウェブサービス版よりも機能が先行していない場合があると作者が述べているため、最新の体験を求める場合はまず公開されているウェブプラットフォームを試すのが良さそうです。

まとめ

Edit-Banana は、情報の「静的なフリーズ状態（画像化）」を解除し、再び「動的な編集状態」へと戻すマジックのようなツールです。図表作成というクリエイティブな作業において、過去の資産を無駄にせず、スムーズに再開できるようにするこのフレームワークは、多くの資料作成現場で強力な武器になるでしょう。
