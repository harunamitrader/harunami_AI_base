これは何か

claw-code は、Claude Code 系のエージェントハーネス設計を観察しながら、公開可能な形で再構成していくためのリポジトリです。README では、現在の主軸を Python ベースのクリーンルーム再実装としつつ、より高速で安全な実行基盤として Rust 版の統合も進めていると説明しています。

何ができるか

現状の Python ワークスペースでは、マニフェスト出力、サブシステム一覧、コマンドとツールの棚卸し、要約表示、パリティ監査といった確認系の CLI をまとめて扱えます。テスト用ディレクトリも分かれており、完全互換のランタイムというより、構造理解と段階的移植を支える作業場として見るのが自然です。

目立つポイント

目立つのは 3 点です。1 つ目は src/ と tests/ を中心にした Python-first の構成。2 つ目は README 上で「元の露出スナップショットを追跡対象から外した」と明示し、現在の公開リポジトリを再実装側に寄せている点。3 つ目は dev/rust ブランチで Rust 版統合を進め、将来的な高速化とメモリ安全性を前面に出している点です。

セットアップや使い方の流れ

README で案内されている入り方は素直です。まず python3 -m src.main summary や manifest で現在の移植状況を確認し、必要なら commands や tools サブコマンドで棚卸しを見ます。その後、python3 -m unittest discover -s tests -v で検証し、ローカルに条件が揃っていれば parity-audit で差分確認を進める流れです。

どんな人向けか

AI エージェントのツール接続やタスク実行のハーネス設計を研究したい人、Codex や OmX を使ったオーケストレーション実例を追いたい人、既存ツールの動きをクリーンルームに寄せて理解したい人に向いています。逆に、すぐ商用運用に入れたい完成品 CLI を探している場合は、現時点では少し研究寄りです。

注意点

README 自体が、元のシステムとの完全な runtime parity にはまだ達していないと書いています。また、出発点に法的・倫理的な議論を伴う文脈があるため、導入判断では README の Disclaimer や背景説明まで読むべきです。公開リポジトリは再実装に寄せられていますが、そこを雑に読み飛ばすと誤解しやすいタイプのプロジェクトです。

まとめ

claw-code は、派手な話題性よりも「いま公開されている実体は何か」を押さえて読むと価値が見えます。現時点では Python で構造を揃えながら検証コマンドを整備し、次段として Rust 実装を育てている段階なので、完成品というより移植プロセスそのものを観察するためのリポジトリとして面白いです。
