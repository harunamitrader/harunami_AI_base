これは何か

autoagent は、agent を動かすのではなく、その harness 自体を meta-agent に改良させていくための実験基盤です。README の一文でも「agent engineering を overnight で自律反復させる」と説明されており、system prompt、tools、agent configuration、orchestration を変更して benchmark score を上げ続ける設計です。

何ができるか

人間は program.md に目的を書くと、meta-agent が現在の harness を読み、benchmark を回し、失敗を診断し、agent.py を修正し、スコアが上がるかを確認して改善を続けます。つまり「agent を使ってタスクを解く」より一段上の、「agent の作り自体を最適化する」ワークフローです。

目立つポイント

特徴は、harness under test をほぼ 1 ファイルの agent.py に寄せ、meta-agent の編集面を明確にしている点です。Harbor 互換 task format と score-driven loop を採用しているので、修正の良し悪しを数値で残しながら hill-climb できるのが分かりやすいです。

セットアップや使い方の流れ

Quick Start は、uv sync、.env 設定、Dockerfile.base から base image を build、tasks/ に Harbor 形式の task を置いて benchmark を回す流れです。その上で coding agent に Read program.md and let's kick off a new experiment! と渡すと、meta-agent が評価と修正を反復する仕組みになっています。

どんな人向けか

prompt tuning や tool 設計を手で詰めるより、評価ループごと自動化したい人に向いています。特に Harbor task を既に持っていて、agent harness を benchmark 駆動で改善したい研究者や agent 開発者にはかなり相性が良さそうです。

注意点

README にもある通り、Docker、Python 3.10+、uv、そして benchmark task の自前準備が必要です。repo 単体では task が同梱されていないので、すぐに完成品を使うというより、自分の評価環境を持っている人向けと考えた方がよいです。

まとめ

autoagent は、AI agent を 1 回走らせるツールではなく、agent engineering の改善ループそのものを automation したい人向けの repo です。meta-agent に harness 改良を任せる発想に興味があるなら、かなり面白い土台です。
