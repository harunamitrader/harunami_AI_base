これは何か

gbrain は、Y Combinator の会長である Garry Tan 氏が公開した、AI エージェントのための「外部脳」構築フレームワークです。メール、カレンダー、会議の議事録、SNS、そして自分自身のメモやアイデアを一箇所に集約。AI エージェントがそれらを「読み・書き」し続けることで、あなたについて世界で最も詳しいパーソナル・アシスタントを育てるための基盤を提供します。

何ができるか

  • 長期記憶の構築： 会話のたびに情報を蓄積し、次にその人物やトピックが登場した際に、過去の経緯を完璧に踏まえた回答が可能になります。
  • インテリジェント・検索（RAG）： ベクトル検索とキーワード検索を組み合わせたハイブリッド検索により、膨大な個人データから瞬時に必要な情報を引き出します。
  • 広範なデータ連携： X（Twitter）、メール、カレンダー、音声通話（Twilio）などのデータを AI エージェントが自らインポート・学習する「レシピ」が用意されています。
  • MCP (Model Context Protocol) 対応： Claude Code や Cursor などの MCP 対応エージェントから、30 以上のツールを通じて脳内の知識にアクセスできます。
  • Local-First ＆ Scalable： 最初はローカルの PGLite（サーバー不要の Postgres）で開始し、規模が大きくなれば Supabase などのクラウド DB へ容易に移行できます。

目立つポイント

単なる「ツール」ではなく、「AI と共に生きるためのライフスタイル（Playbook）」を提案している点が非常にユニークです。特に GBRAIN_SKILLPACK.md と呼ばれるドキュメントには、AI エージェントが「いつ情報を確認し、いつ自分の脳を更新すべきか」という具体的な行動指針が記されています。これにより、AI はただ質問に答えるだけの存在から、自律的にあなたの記憶を整理し、朝起きるたびに賢くなっているパートナーへと進化します。

セットアップや使い方の流れ

  1. リポジトリを git clone し、bun install でセットアップを行います。
  2. gbrain init を実行してローカルに脳（データベース）を作成します。
  3. gbrain import ~/notes/ などで、既存のメモやプロジェクト資料を一括で読み込ませます。
  4. Claude Code などのエージェントに MCP 設定を追加し、エージェントが gbrain のスキルセットを使えるようにします。
  5. あとは普段通り AI と会話するだけ。エージェントが自動でエンティティを検出し、情報の紐付け（バックリンク）や更新を行ってくれます。

どんな人向けか

  • 大量の会議、メール、人脈を抱え、AI に自分の文脈を完璧に理解させたい経営者やリーダー。
  • 情報の断片が散らばり、自律型 AI エージェント（OpenClaw 等）の長期記憶基盤を探している開発者。
  • 「昨日の続き」を完璧に覚えてくれている、真のパーソナル AI を育てたいと願うすべての人。

まとめ

gbrain は、AI が「あなたという人間」についての知識を複利（Compounding）で蓄積していくための土壌です。使えば使うほど、AI はあなたにとって代えがたい存在へと成長します。AI エージェントという新しいインターフェースに、自分だけの歴史と文脈を吹き込みたい人にとって、これ以上ない強力な武器となるでしょう。
