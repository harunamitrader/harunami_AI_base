AI エージェントに長期・短期記憶を授ける、超高速・高スケーラビリティなメモリエンジン。

記事作成日 2026-03-26
作成時順位 5位
対象 repo supermemoryai/supermemory

これは何か

supermemory は、AI アプリケーションやエージェントが、過去の対話やユーザーの好み、外部ドキュメントを「記憶」として保持・検索するための、AI 時代に特化したマネージド・メモリ基盤です。

何ができるか

会話の中からユーザーの個人的な好み（User Profile）を抽出し、時間が経過しても忘れず、矛盾があれば自動更新する動的なメモリ管理を提供します。単なるテキストだけでなく、PDF、画像、ドキュメント、そして動画といった多様な形式のインジェストに対応しており、それらを横断して 300ms 未満という驚異的な速さでセマンティック検索（RAG）を実行できます。

目立つポイント

TypeScript で構築された高いスケーラビリティを備え、月に 1,000 億トークン以上の負荷にも耐える設計となっています。主要なメモリベンチマーク（LongMemEval 等）で世界 1 位を記録しており、Google Drive、Notion、GitHub 等との公式コネクタを通じたリアルタイム・シンクロ機能も極めて強力です。

セットアップや使い方の流れ

開発者は Supermemory API を通じてエージェントに記憶機能を接続するか、用意された SDK を利用して既存の AI フレームワークに統合します。「Memory Router」機能を使えば、コードを一行も書かずに特定の LLM 呼び出しに記憶層を挟み込むことも可能です。

どんな人向けか

「一回きり」のチャットボットではなく、何ヶ月も使い続ける中で自分を理解してくれるパーソナル・アシスタントを作りたい開発者や、組織内の膨大なドキュメントを常にフレッシュな状態で AI に参照させたいエンタープライズユーザーに適しています。

注意点

記憶の規模が大きくなるにつれ、データの鮮度管理（新しい情報が古い情報を上書きする条件など）のチューニングが重要になります。プライバシー設定やメタデータによるデータ分離を正しく設定しないと、意図しない文脈が呼び出される可能性があるため、設計時にはフィルタリング機能の活用が推奨されます。

まとめ

AI の「知能」は LLM によって解決されつつありますが、真の有用性は「記憶」にかかっています。Supermemory は、その欠落していた記憶層を極めて高いレベルで埋める、インフラ級のプロダクトです。
