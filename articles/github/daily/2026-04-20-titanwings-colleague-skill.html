これは何か

colleague-skill（移行中：dot-skill）は、特定の人格や思考プロセスを「AIスキル」として抽出・再現（蒸留）するためのオープンソース・フレームワークです。 離職した同僚、卒業したメンター、さらには著名人や架空のキャラクター、あるいは自分自身など、対象に関連するテキストデータからその人物らしい反応をするエージェントを生成します。

何ができるか

  • 人格の蒸留（Distillation）: チャットログ、メール、インタビュー記事、動画の書き起こしなどを読み込み、対象の「思考の枠組み」と言動の特徴を学習したスキルを生成します。
  • 3つのファミリー対応:
    • Colleague: プロフェッショナルなスキル、特定のドメイン知識、仕事の進め方を再現。
    • Relationship: 家族や友人など、感情や親密な関係性の文脈を考慮。
    • Celebrity: 著名人の思考モデルや意思決定プロセスを分析・再現。
  • 主要Agentへの統合: Claude Code, Codex, Hermes, OpenClaw などのエージェントホストに直接インストールして、「/dot-skill」コマンドで呼び出すことができます。

目立つポイント

  • 多角的な分析エンジン: 単なるパロット（オウム返し）ではなく、作品・インタビュー・意思決定・表現の癖・外部の評価・タイムラインの6次元から対象を分析します。
  • 動的な自己進化: 新しいファイルを追記することで差分を自動解析し、既存の結論を上書きせずにマージ（Evolution）する機構を備えています。
  • 対話による修正: AIの反応に対して「彼ならそんなことは言わない」とフィードバックを送ることで、即座に人格（Persona）を微調整可能です。

セットアップや使い方の流れ

  1. インストール: 対応するエージェントホスト（Claude Code等）でリポジトリのURLを渡し、自動セットアップを実行します。
  2. 初期化: /dot-skill コマンドを実行し、蒸留したい対象の「ファミリー（同僚/関係/著名人）」を選択します。
  3. データ提供: 対象のプロフィール、特徴タグ、およびデータソース（チャットログやメールの書き出し等）を流し込みます。
  4. スキルの使用: 生成されたスキル /{character}-{slug} を呼び出し、彼／彼女と対話を開始します。

どんな人向けか

  • 退職や卒業によって失われる「仕事の勘どころ」や「属人的な知恵」をデジタル化して残したい開発チーム。
  • 憧れの思想家や天才的なエンジニアなら、自分の直面している問題にどう答えるかを身近に感じたい学習者。
  • 「デジタル生命」や「人格の再現」というSF的な試みを、実務に近いエージェント環境で試してみたい技術者。

注意点

  • データの質が重要: 蒸留の精度は入力データの「密度」と「質」に大きく左右されます。表面的な会話だけでなく、意思決定の背景がわかるデータが理想的です。
  • 倫理とプライバシー: 実在の人物を再現する場合、そのデータの取り扱いやAI化の是非については周囲の同意や倫理的な配慮が不可欠です。

まとめ

「離別を温かなスキルに変える」というコンセプトを掲げる colleague-skill は、AI時代のナレッジマネジメントとパーソナライズされたAIの究極の形を提示しています。 単なるチャットボットを超え、特定の人物の「思考の癖」を道具として使えるようにするという発想は、今後の開発チームの在り方に大きな影響を与えるかもしれません。
