これは何か

openagents は、複数の AI エージェントを 1 つのブラウザベース workspace に集約して協調させるためのオープンソース基盤です。README では OpenAgents Workspace として紹介されており、エージェントがどこで動いていても 1 つの URL から管理し、会話させ、共同作業させることを狙っています。

何ができるか

Claude Code、OpenClaw、Codex CLI、Cursor など対応エージェントを同じ workspace へ接続し、同じ会話スレッド、ファイル、ブラウザを共有できます。@ メンションでエージェントを呼び分けたり、共有ブラウザ上でページ操作やスクリーンショット取得を行わせたり、ファイルをアップロードして人とエージェントで共通利用したりできます。

目立つポイント

特徴は「マルチエージェント」だけでなく、「persistent hub」として workspace URL を持つ点です。README の説明では Slack のような場所として扱われており、別マシンや別ターミナルで動くエージェントを同じ文脈へ集められます。加えて launch week 時点で launcher、CLI、desktop app が揃っており、単発デモではなく日常運用を狙った形です。

セットアップや使い方の流れ

README の quick start はかなり短く、macOS / Linux なら install script、Windows なら PowerShell スクリプトを叩いて、その後に agn を実行する流れです。CLI 以外に desktop app もあり、既存エージェントを workspace へ接続してからブラウザで管理するのが基本の使い方になります。

どんな人向けか

複数の AI エージェントを並行で動かしていて、進捗確認や共同作業の文脈が散らばって困っている人に向いています。特にインフラ担当エージェント、開発エージェント、ユーザー対応エージェントのように役割を分けている場合は、1 つの workspace に寄せる価値が大きそうです。

注意点

リポジトリは大きく、workspace、launcher、SDK、各 agent 連携まで含むため、導入対象を絞らずに触ると把握に時間がかかります。また「どの agent をどの権限でつなぐか」の設計は別途必要なので、まずは 1 から 2 種類のエージェントで小さく試す方が安全です。

まとめ

openagents は、AI エージェントを増やした後に生まれる「文脈が散る」「連携が面倒」という問題を、共有 workspace でまとめて解こうとする基盤です。マルチエージェント運用を本気で回したい人ほど、触る価値があるリポジトリです。
