マイグレーションやローカル環境の実行、エッジ関数のデプロイを管理する最新のSupabase公式CLI。

記事作成日2026-03-29
作成時順位ピックアップ
対象 reposupabase/cli

これは何か

オープンソースのFirebase代替として名高いBaaS「Supabase」の開発を、ローカル環境やターミナルから一元管理するための公式コマンドラインツールです。

何ができるか

ローカルでのSupabase環境のコンテナ起動、本番環境へのPostgreSQLマイグレーションの実行、データベーススキーマからの型生成、さらにエッジ関数（Edge Functions）のデプロイやバックアップなど、開発サイクルのほぼすべてをカバーします。

目立つポイント

BaaSでありながら「開発はローカルで行う」というモダンなワークフローとの親和性が非常に高く、フロントエンドエンジニアが複雑なバックエンドインフラを簡単にコードベースで操作・管理できる設計が光ります。

セットアップや使い方の流れ

npmパッケージやHomebrew等経由でCLIをインストールし、プロジェクトディレクトリで初期化（init）と起動（start）を行うだけで、ローカル環境に完全なSupabaseスタックが立ち上がり開発を始められます。

どんな人向けか

Supabaseを使ってアプリケーションを構築しているすべての開発者や、データベースのバージョン管理（マイグレーション）などを自動化・安全に行いたいチーム開発向けです。

注意点

ローカル実行環境はDockerコンテナに依存しているため、CLIをフル活用してスタックを立ち上げる前提として、安定したDockerの稼働環境が開発機に必須となります。

まとめ

フルスタック開発の流れを大幅に最適化し、Supabaseの真価をローカルから最大限に引き出すための欠かせない開発基盤ツールです。
