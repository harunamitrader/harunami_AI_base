これは何か

Docker コンテナ内で動作する仮想ブラウザを、WebRTC テクノロジーを用いてリモートから操作・閲覧できるオープンソースのセルフホスト型プラットフォームです。自分のサーバーに閉じた環境でブラウザを走らせ、特定の URL を複数人で同時に見たり、制御権を渡して共同作業を行ったりすることができます。かつて人気だった Rabb.it などのウォッチパーティーサービスのセルフホスト版として、強力な支持を得ているプロジェクトです。

何ができるか

  • WebRTC による超低遅延： 画面の書き換えを WebSockets などの静止画転送ではなく WebRTC ストリーミングで行うため、動画再生も極めてスムーズです。
  • 共同ブラウジング： 同時に複数のユーザーがルームに入り、同じブラウザ画面を共有できます。チャット機能や、マウス・キーボードの制御権をワンクリックでリクエスト/受け渡しする機能を備えています。
  • 多様なアプリケーション： デフォルトの Chromium や Firefox 以外にも、VLC やデスクトップ全体（XFCE 等）をコンテナ内で起動して共有することも可能です。
  • プライバシーとセキュリティ： セルフホスト型のため、履歴やクッキーなどの機密データはすべて自分の管理下にある環境に留まり、クライアント（閲覧者）側には動画ストリームのみが送信されます。
  • 自動化連携： API や neko-rooms を活用することで、オンデマンドで仮想ルームを作成・破棄するような高度な運用も可能です。

目立つポイント

単なるリモートデスクトップ（VNC/RDP）とは異なり、ブラウザ内に閉じた「ウェブアプリ」として構築されているため、特別なクライアントソフトなしに高品質なオーディオ・ビデオ共有ができる点が非常に優秀です。また、VR Chat のワールド内にブラウザを持ち込んで友達と動画を見るなど、クリエイティブな用途にも幅広く使われています。

セットアップや使い方の流れ

  1. docker-compose.yaml を用意し、m1k1o/neko:chromium などのイメージを指定して起動します。
  2. ブラウザで指定したポート（デフォルト 8080 等）にアクセスし、ルーム名とパスワードを設定。
  3. ルームに入ると、コンテナ内で動いているブラウザが表示されます。画面右側のパネルから、制御権の切り替えやチャット、設定変更が行えます。
  4. 他のユーザーを招待して、一緒に動画を見たり、サイトのデバッグをしたりして活用します。

どんな人向けか

  • 離れた場所にいる友達や家族と、同期ズレなしに高画質でアニメや映画を鑑賞したい人。
  • 開発チームで、OS や環境に依存しない「共有された開発用ブラウザ」を使ってペアプロやデバッグを行いたいエンジニア。
  • サーバー上で動作する常時起動ブラウザを確保し、どこからでも自分の環境にアクセスしたいパワーユーザー。
  • VR 空間や特定のウェブページ内に、インタラクティブな仮想ブラウザ機能を埋め込みたい開発者。

注意点

WebRTC を使用するため、サーバーに一定以上の帯域と、UDP ポートの開放（NAT 越えのための STUN/TURN 構成が必要な場合あり）が求められます。また、複数のユーザーで同時に動画をストリーミングする場合、サーバー側の CPU リソースもそれなりに消費します。

まとめ

neko は、「ブラウザを共有する」というシンプルながら強力な体験を、セルフホストという究極の自由度で提供してくれます。ウォッチパーティー、共同作業、リモートデバッグ、プライベートブラウジングなど、その使い道はユーザーの想像力次第で無限に広がります。
