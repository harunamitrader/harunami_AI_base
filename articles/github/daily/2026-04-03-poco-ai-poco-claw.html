これは何か

poco-claw は、OpenClaw の代替を狙った Web ベースの AI agent アプリです。README では「safer, more beautiful, and easier-to-use alternative to OpenClaw」とされており、Claude Code ベースの agent を土台にしつつ、より整った UI と安全な実行環境を前面に出しています。

何ができるか

browser 内蔵の自律リサーチ、project management、conversation queueing、file uploads、GitHub repo integration、background execution、scheduled triggers まで一通り備えています。さらに DingTalk、Feishu、Telegram への IM 連携と push 通知もあり、ブラウザを閉じた後も agent を回し続ける使い方が想定されています。

目立つポイント

目立つのは sandboxed runtime を明示している点と、Artifacts view / Playback view のような UI 側の完成度です。HTML、PDF、Markdown、画像、動画などをその場でプレビューでき、コマンド I/O や browser session、Skills / MCP 呼び出しの再生もできるので、単なる chat UI よりかなり運用寄りです。

セットアップや使い方の流れ

Quick Start はかなり短く、./scripts/quickstart.sh を実行して設定生成とサービス起動を済ませ、その後 http://localhost:3000 を開く流れです。README では one-click Docker deployment と書かれており、詳細は別の Deployment Guide に逃がしているので、まずローカルで立ち上げてから必要に応じて self-hosting を詰める形になります。

どんな人向けか

OpenClaw 系の agent UI を試したいが、もっと見やすい管理画面や IM 連携、安全な実行分離を重視したい人に向いています。特に browser research、MCP、Skills、遠隔通知まで 1 つの画面に寄せたい人には相性が良さそうです。

注意点

README 上でも cloud subscription は coming soon で、現状は self-host 前提の色が強いです。また機能が多く、backend、executor、frontend、browser、IM 連携まで含む構成なので、最初から全部を使おうとせず、まずはローカルの quickstart で基本フローを掴む方が安全です。

まとめ

poco-claw は、Claude Code 系 agent を Web UI と sandbox で扱いやすく包み直した実用志向のプロジェクトです。OpenClaw 系の使い勝手に不満がある人ほど、一度触る価値があります。
