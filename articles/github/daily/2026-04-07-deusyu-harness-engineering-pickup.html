これは何か

harness-engineering は、OpenAI が提唱した AI エージェントの自律的な動作を支えるための設計思想「ハーネス・エンジニアリング」を、概念から実装まで体系的にまとめたリソース集です。 エージェントがただ「賢い」だけでなく、現実のタスクを安全・確実に完遂できるよう、周囲の制御機構（ハーネス）をいかに構築すべきかを提案しています。

何ができるか / 構成要素

  • Ralphループによる自律完遂: タスク達成までエージェントを自律的に繰り返し実行させる基本パターンの実装例。
  • サンドボックスと制御: エージェントを隔離しつつ外部ツールへのアクセスを管理する、安全な実行環境の構築。
  • Backpressure Gates（背圧ゲート）: エージェントによる過剰な API 消費や無限ループを防ぐための制御機構。
  • エージェント可読性の設計: エージェントが自身の状態（ステータス、ログ、コンテキスト）を正確に把握し、内省（Self-Correction）しやすくするための設計手法。

目立つポイント

最大の強みは「プロンプトエンジニアリング」の先にある **「システムとしてのエージェント運用」** に焦点を当てている点です。 単発の指示ではなく、数時間に及ぶ長期的なタスクを自律的に実行させる際のリスク管理（コスト、安全性、リソース）をエンジニアリングとして解決するための知見が凝縮されています。

セットアップや活用の流れ

  1. リポジトリ内のロードマップを読み、ハーネスの概念（Ralphループ等）を理解します。
  2. 付属の Bash スクリプトや実装例（Rust 等）を参考に、自律型エージェントの実行基盤を作成。
  3. エージェントに「ハーネス」という制御盤を通した道具アクセスを与え、安全性を担保しながら自律的に動かします。
  4. 永続的な状態管理（State Machine）を組み込み、長期タスクでのレジューム機能を実装します。

どんな人向けか

  • Claude Code などの自律型エージェントを、より大規模・実用的なタスクに適用したい開発者。
  • エージェントの「暴走」や「APIコストの浪費」を防ぎつつ、自律性を最大化したいシステム設計者。
  • AI エージェントを単なるチャット相手ではなく、信頼できる「自律的なチームメンバー」として構築したい人。

注意点

本リポジトリは概念的・実験的な内容を多く含みます。実際のプロダクション環境に適用する際は、各プラットフォーム（Docker 等）でのセキュリティ設定やリソース制限など、物理的な制約も適切に考慮する必要があります。

まとめ

harness-engineering は、AI エージェントを「暴れ馬」にしないための手綱（ハーネス）を設計する学問です。 今後、エージェントがより自律的に動くようになる時代において、この「制御の設計」は、プロンプトの記述以上に重要なスキルになっていくでしょう。
