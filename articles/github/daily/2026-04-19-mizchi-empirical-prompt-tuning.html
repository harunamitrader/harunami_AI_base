これは何か

日本の技術コミュニティのフロントランナーである mizchi 氏が公開した、Claude Code (AIエージェント) 用の高度なシステムプロンプト（スキル）です。 「プロンプトの良し悪しは書き手には客観視できない」という前提に立ち、 別の AI エージェント（Subagent）に実際にプロンプトを実行させ、その結果をメトリクスと質的フィードバックの両面から評価・反復改善するためのフレームワークを定義しています。

何ができるか

  • バイアス排除読み： 指示を「白紙」の Subagent に読ませ、書き手の思い込みを排除。
  • 定量的メトリクス測定： 成功率、精度（％）、ステップ数（tool_uses）、所要時間、再試行回数を数値化。
  • 質的フィードバック収集： Subagent から「不明瞭点（解釈に迷った箇所）」や「裁量補完（指示がないため勝手に決めた箇所）」を収集。
  • 収束判定： 2回連続で不明瞭点ゼロ、かつ改善が飽和するまで反復を自動継続。
  • 過適合チェック： 未知の Hold-out シナリオを用いた最終検証機能。

目立つポイント

**「AI に AI のための説明書をテストさせる」** という、モデルの知能を最大限に活用した設計が秀逸です。 プロンプトを「文学」ではなく「ソフトウェア」として扱い、TDD（テスト駆動開発）のようなサイクルで堅牢化していくアプローチは、 実用レベルの AI エージェントを運用する上で不可欠な視点を提供しています。 単に精度 100% を目指すだけでなく、`tool_uses`（ステップ数）の偏りから「エージェントの迷い」を検出し、 構造的な欠陥（指示の不備）を炙り出す仕組みも非常に実践的です。

セットアップや使い方の流れ

mizchi 氏の chezmoi-dotfiles 内にある dot_claude/skills/ 以下のファイルを自身の Claude Code プロジェクト（通常は .claude/skills/）にコピーします。 改善したいプロンプトと評価シナリオ、要件チェックリストを用意し、Claude Code に「本スキルに従ってプロンプトをチューニングして」と指示することで、 Subagent を駆使した反復改善プロセスが開始されます。

どんな人向けか

Claude Code の挙動が不安定で悩んでいる開発者、あるいは頻繁に使用する重要なカスタムスキル（Slash Command）を、 極限まで堅牢かつ効率的に（認知負荷を低く）磨き上げたいプロンプトエンジニアに最適です。

まとめ

empirical-prompt-tuning は、AI エージェント時代の「デバッグと最適化」の新しい標準を示すツールです。 人の主観を排し、バイナリのようにプロンプトを精査するこの手法は、 「AI への指示の曖昧さ」を徹底的に排除したいすべての開発者にとって、最高のガイドラインとなるでしょう。
