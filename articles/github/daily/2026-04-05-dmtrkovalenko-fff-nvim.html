これは何か

fff.nvim は、Neovim 向け fuzzy finder plugin でありつつ、AI agent 向け MCP としても使える file search toolkit です。README では「速い検索」だけでなく、履歴や frecency を使って無駄な探索を減らす点が前面に出ています。

何ができるか

ファイル検索、grep、glob、multi-grep を 1 つの操作系にまとめ、typo に強い fuzzy match と git 状態、定義一致、ファイルサイズなどを加味した順位付けを返せます。AI agent 向けには MCP 経由で接続し、少ない往復で目的ファイルへ寄せる設計です。

目立つポイント

workspace は fff-core、fff-mcp、fff-nvim、fff-grep などに分かれており、Neovim plugin だけで終わらず検索基盤を再利用できる構成です。Rust 実装に寄せたことで大規模 repo でも速度を出しやすく、MCP と editor plugin が同じ検索体験を共有できます。

セットアップや使い方の流れ

AI agent 側は install script で MCP を入れ、Claude Code や Codex に接続します。Neovim 側は lazy.nvim か vim.pack で導入し、初回に binary を取得または build してから find_files や live_grep を使う流れです。

どんな人向けか / 注意点

巨大 repo を頻繁に触る Neovim ユーザー、agent harness の file search を高速化したい人、grep と fuzzy finder を分断したくない人に向いています。一方で README でも beta 的な最適化余地や debug 表示を前提にしており、安定運用前にキー設定や検索モードの癖を確認した方が安全です。

まとめ

fff.nvim は、editor plugin と agent tool の間にある検索レイヤーを 1 つに寄せた実務寄りの project です。2026-04-05 時点で Trending Rank 5 に入っているのは、agent 時代に「速い file search」自体が独立した価値になっていることを示しています。
