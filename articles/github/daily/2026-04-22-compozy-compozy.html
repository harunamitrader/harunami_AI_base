Compozy

40種以上のAIエージェントをオーケストレーション。アイデアからPRD作成、タスク分割、コード実装、レビューまでを統合管理。

これは何か

AI支援開発の全ライフサイクルを一つのパイプラインで駆動するためのオーケストレーションツールです。Claude Code、Cursor、Gemini といった40種類以上のAIエージェントやIDEを統合し、「アイデアの具体化」から「実装」、「自動レビュー」までを一気通貫で管理します。

何ができるか

「アイデア → PRD → 技術仕様書 → タスク分解 → 並列実装 → PRレビュー」という一連のフローを自動化できます。各ステップで生成されるドキュメントはすべて Markdown 形式で、バージョン管理しやすく、人間が介在して修正を加えることも容易です。

目立つポイント

単一のエージェントに頼るのではなく、複数のエージェントを並列に走らせてタスクを消化する「マルチエージェント実行」や、既存のコードベースを解析して各タスクに最適なコンテキストを付与する「コードベース理解（Enrichment）」機能が非常に強力です。また、エージェントの設定を .compozy/agents/ 配下にパッケージ化して再利用できる柔軟性も備えています。

セットアップや使い方の流れ

Go製のシングルバイナリとして提供されており、Homebrew や NPM を通じて導入可能です。導入後は compozy setup を実行することで、お使いの AI エージェント（Claude Code 等）に Compozy 共通のスキル（プロンプトセット）をインストールして使い始めることができます。

どんな人向けか

Cursor や Claude Code などの AI ツールを頻繁に利用しているが、それらをバラバラに使うのではなく、一つのプロジェクトとして体系的に管理したいエンジニアに最適です。特に大規模な機能追加や、要件定義からしっかりと固めて進めたいプロジェクトの「司令塔」として機能します。

注意点

ワークフローの各段階で生成される Markdown ファイルは、次のフェーズの「設計図」となります。そのため、最初の「アイデア」や「PRD」の段階で方向性がズレると、その後の実装も引きずられてしまうため、フェーズごとの人間によるレビュー（チェック）が推奨されます。

まとめ

「プロンプトを投げて終わり」ではなく、開発プロセスそのものを AI と共に構築・洗練していくためのプラットフォームです。AI 時代のソフトウェア開発における「新しい標準ワークフロー」を先取りできる、野心的なプロジェクトです。
