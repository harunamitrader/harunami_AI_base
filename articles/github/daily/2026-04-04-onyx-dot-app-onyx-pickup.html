これは何か

onyx は、LLM 向けの application layer を自前で持てるオープンソース AI プラットフォームです。README でも Open Source AI Platform と明示されており、単なる chat UI ではなく、RAG、web search、agent、code execution、artifacts、MCP までまとめて self-host できるのが軸です。

何ができるか

50 以上の connector による indexing、Agentic RAG、Deep Research、Custom Agents、Web Search、Artifacts、Actions & MCP、Code Execution、Voice Mode、Image Generation までを一つのプロダクトで扱えます。さらに主要な商用 LLM と self-hosted LLM の両方を接続できるため、モデル選択の自由度も高いです。

目立つポイント

目立つのは「AI 機能を足した社内検索」ではなく、かなり広い機能を 1 つの基盤へ寄せている点です。README では Lite mode と Standard mode を分けており、軽量な chat / agents 用の導入から、大規模運用向けの indexing、job queue、Redis、MinIO を含む構成まで段階的に選べます。

セットアップや使い方の流れ

最短導入は README にある curl -fsSL https://onyx.app/install_onyx.sh | bash の 1 行です。加えて Docker、Kubernetes、Helm、Terraform、主要クラウド向け guide も docs に用意されています。まず Lite mode で chat / agents を試し、必要なら Standard mode に広げる流れが現実的です。

どんな人向けか

社内向け AI アプリを self-host したいチーム、あるいは RAG と Agents を同時に扱いたい開発組織に向いています。特に「複数の LLM ベンダーを切り替えつつ、検索、調査、実行、共有まで一体で持ちたい」という用途ではかなり相性が良さそうです。

注意点

README の Standard mode は検索インデックス、background workers、cache、blob store まで含むため、運用は軽くありません。小さく始めるなら Lite mode から入るべきです。また Enterprise 向け機能や Cloud も用意されているので、OSS 部分と商用拡張の境界は導入前に確認した方が安全です。

まとめ

onyx は、RAG ツール、agent platform、research assistant を別々に組み合わせる代わりに、1 つの self-host 可能な基盤へまとめたい人向けの大型 OSS です。AI アプリ基盤を自前で持ちたいチームなら、かなり有力な選択肢です。
