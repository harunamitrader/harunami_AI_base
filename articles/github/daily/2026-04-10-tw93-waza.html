これは何か

Waza（技） は、エンジニアが長年の経験で培った「良い習慣」を、AI エージェント（特に Claude Code）が実行可能な「スキル」としてまとめたリポジトリです。「AI は作業を速くするが、思考を深くするわけではない」という哲学のもと、AI にエンジニアらしい振る舞いをさせるためのプレイブックを提供します。

何ができるか

Claude Code にスラッシュコマンド（/think, /design 等）を追加することで、特定のワークフローを AI に強制または推奨できます。また、コンテキストの使用量やクォータを視覚化するステータスライン、英文ライティングのコーチング機能なども含まれています。

目立つポイント

  • 厳選された 8 つのスキル: /think（思考）, /design（設計）, /check（レビュー）, /hunt（検索）, /write（執筆）, /learn（学習）, /read（読解）, /health（監査）の 8 種に絞ることで、学習コストを抑えつつ実戦的な支援を実現。
  • 軽量でミニマル: 既存の巨大なスキル集（Superpowers 等）と比較して、設定がシンプルで「道具としての使い勝手」を重視しています。
  • 失敗から学んだ「落とし穴」集: 各スキルには、実際のプロジェクトで発生した失敗事例に基づいた「Gotchas（注意点）」セクションが含まれており、AI が同じ過ちを繰り返さないよう工夫されています。
  • 開発体験の向上: ターミナル上で一目で状況がわかるステータスラインや、自然な英語表現を学べるコーチング機能など、細かな配慮が行き届いています。

セットアップや使い方の流れ

Claude Code がインストールされている環境であれば、以下の 1 コマンドで全てのスキルを導入できます： 

npx skills add tw93/Waza -a claude-code -g -y

個別のスキルのみを導入したり、他のエージェント（Cursor 等）のコンテキストとして利用することも可能です。

どんな人向けか

  • Claude Code をメインの相棒として使いこなしたい人: 独自のコマンドで開発フローを標準化したい場合に最適です。
  • AI の「出力品質」に物足りなさを感じているエンジニア: AI に「プロらしい思考プロセス」を挟ませることで、精度の向上を狙えます。
  • 英語でのやり取りを改善したい非ネイティブ開発者: コーチング機能により、やり取りを通じて自然に英語力を磨くことができます。

注意点

一部の機能（/health や /read での自動スクリプト実行など）は、Claude Code 以外のプラットフォームでは制限される場合があります。また、破壊的な Git コマンドを拒否リストに追加するなどの、安全性のための設定が推奨されています。

まとめ

Waza は、単なるプロンプト集ではなく、エンジニアの「魂」を AI に吹き込むためのツールキットです。モデルの進化を待つだけでなく、モデルが「どう考えるべきか」を制約と習慣によってデザインすることで、より健全で高品質な開発体験を構築できます。
