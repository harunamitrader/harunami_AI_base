これは何か

AI 研究者の Andrej Karpathy 氏が提唱した「LLM Wiki」の概念（LLM にドキュメントを読み込ませ、構造化された Wiki を自動構築・保守させる手法）を、一般ユーザーが利用可能なデスクトップアプリケーションとして実装したプロジェクトです。Tauri v2 と React をベースにしたクロスプラットフォーム対応アプリであり、単なる RAG（検索・回答）を超えて、持続的で閲覧性の高い知識ネットワークを構築します。

何ができるか

  • 2ステップ・インジェスト： LLM による高度な解析（Chain-of-Thought）を経てから Wiki ページを生成することで、ソースの追跡可能性（Traceability）と高品質な要約を両立。
  • インテリジェント・グラフ： Louvain アルゴリズムを用いて知識のクラスタを自動検出し、視覚的なナレッジグラフとして表示。
  • Deep Research： 構築された Wiki 内の知識の欠落（Knowledge Gap）を AI が分析し、Web 検索（Tavily API）を用いて自動的に情報を補完。
  • Web クリッパー： Chrome 拡張機能経由でブラウザから直接、整理された情報としてナレッジベースに流し込み。
  • Obsidian 連携： 生成されるデータは Markdown 形式であり、Obsidian の Vault としてもそのまま利用可能。

目立つポイント

従来の RAG システムが「質問のたびに検索する」使い捨てのプロセスになりがちだったのに対し、本プロジェクトは「インクリメンタルに Wiki を育てる」点に注力しています。知識の SHA256 キャッシュによる効率的な読み込みや、同期・非同期のレビューシステムなど、実用的な知識管理ツールとしての機能が詰め込まれています。

セットアップや使い方の流れ

  1. リポジトリの Releases ページから、OS（macOS, Windows, Linux）に合わせたバイナリをダウンロードしてインストール。
  2. 設定画面で OpenAI や Anthropic などの API キーと使用モデル（DeepSeek、QwQ などの思考型モデルも推奨）を設定。
  3. ドキュメント（PDF, DOCX, MD 等）をインポート。Activity Panel で AI による解析・自動構築が進むのを確認します。
  4. 構築された Wiki をチャットで検索したり、ナレッジグラフで繋がりを探索したりして活用します。

どんな人向けか

  • 溜まり続けるドキュメントや Web 記事を、読みやすく整理された「自分専用の百科事典」に変えたい人。
  • RAG を自作したいが、デスクトップアプリとしての完成度も求める開発者。
  • Andrej Karpathy 氏の LLM 活用戦略に興味があり、その具現化を体験したいエンジニア。

注意点

インジェスト（読み込み）の過程で大量の LLM トークンを消費する可能性があるため、モデルの選択や設定されたコンテキストウィンドウ サイズには注意が必要です。また、自動生成される Wiki リンクの正確性を保つため、定期的な「Lint（整合性チェック）」機能の実行が推奨されます。

まとめ

llm_wiki は、AI エージェントを「個人の司書」として雇うような体験を提供します。デスクトップアプリとしての利便性と、グラフ理論や最新の LLM 手法を組み合わせた強力な検索・保守性能を兼ね備えており、セカンドブレイン構築の新たなスタンダードになり得るポテンシャルを持っています。
