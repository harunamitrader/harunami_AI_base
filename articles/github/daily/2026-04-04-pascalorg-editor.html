これは何か

editor は、React Three Fiber と WebGPU を使って 3D 建築プロジェクトを編集する Pascal Editor の monorepo です。

README では core、viewer、apps/editor を分け、シーン定義、3D 表示、実際の編集 UI を分離した構成が丁寧に説明されています。

何ができるか

Site / Building / Level / Wall / Zone / Item などのノードで建物を表現し、壁、床、天井、屋根、ゾーン、家具配置まで 3D シーンとして管理できます。

Zustand の各 store、scene registry、dirty node ベースの system 更新で、編集操作と 3D 再計算を分けているのが特徴です。

目立つポイント

単なるデモではなく、ノード階層、イベントバス、空間グリッド、Boolean geometry、undo/redo まで README に整理されており、拡張前提の設計が見えます。

Viewer を既定表示層、Editor をツール拡張層として扱うので、建築編集機能を部分利用したいケースにも流用しやすそうです。

セットアップや使い方の流れ

開発開始はリポジトリのルートで bun install 後に bun dev を実行する想定です。Turborepo の watch と Next.js の editor アプリがまとめて立ち上がります。

本番ビルドや package 公開は turbo build と npm workspace publish を使う流れで、アプリとライブラリを同時に育てる前提です。

どんな人向けか / 注意点

建築・空間設計系の SaaS、3D 編集 UI を持つ業務アプリ、Three.js で大きめの編集体験を作りたいチームに向いています。

一方で技術スタックはかなり厚く、WebGPU 前提や monorepo 前提の理解も要ります。単純な 3D ビューア用途にはオーバースペックです。

まとめ

editor は、建築向け 3D 編集の状態管理と表示責務をきちんと分けた、実装参考価値の高い repo です。2026-04-04 の Trending Rank 4 として見ても、構成そのものを学ぶ価値があります。
