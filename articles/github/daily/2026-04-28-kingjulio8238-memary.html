Memary

自律型AIエージェントに「人間の記憶」を与えるオープンソースのメモリレイヤー。ナレッジグラフを活用し、過去の対話やユーザーの知識レベルを追跡・管理する。

これは何か

自律型エージェントの開発に向けたオープンソースのメモリ・インフラストラクチャです。エージェントが過去のやり取りを「人間の記憶」のように蓄積・整理できるよう、ナレッジグラフ（Neo4jまたはFalkorDB）と連携して情報を構造化し、長期的なコンテキスト管理を実現します。

何ができるか

ユーザーとの対話中に得た情報を自動的に抽出し、エージェントの記憶（グラフデータベース）に書き込みます。この記憶は「Memory Stream（情報への接触履歴のタイムライン）」と「Entity Knowledge Store（特定のエンティティへの参照頻度と最新性）」に分かれて管理され、ユーザーの興味や理解の深さに合わせたパーソナライズされた回答を生成することが可能になります。

目立つポイント

単に過去のチャット履歴をベクトル検索するのではなく、LlamaIndex を用いて複雑なナレッジグラフを構築し、再帰的検索やマルチホップ推論を用いて関連情報を引き出す点が特徴です。これにより、複数要素が絡む高度な質問に対しても、高速かつ正確な文脈の復元が可能です。さらに、マルチグラフ機能により、1つのシステム内で複数のユーザーやエージェントごとの独立した記憶を管理できます。

セットアップや使い方の流れ

Python（3.11以下推奨）環境で `pip install memary` を実行します。`.env` ファイルに OpenAI 等の API キーと、Neo4j または FalkorDB の接続情報を設定します。ローカルで動作する Ollama のモデル（Llama 3 や LLaVA）と統合されており、リポジトリに含まれる Streamlit アプリ（`app.py`）を起動するだけで、記憶機能を持ったエージェントとのチャットUIをすぐに試すことができます。

どんな人向けか

ユーザーの好みを長期間学習し続けるパーソナルアシスタントを開発したいエンジニアや、LLMエージェントに「状態」や「経験」を持たせ、より人間に近い推論を実現（AGIへのアプローチ）したい研究者に最適です。

注意点

フル機能を利用するためにはグラフデータベース（Neo4j/FalkorDB）の用意が必須です。また、現在組み込まれている ReAct エージェントはデモ用のものであり、将来のバージョンではこのエージェント自体は切り離され、任意のフレームワークから呼び出せる純粋なメモリ・モジュールとして独立する予定であることがアナウンスされています。

まとめ

Memaryは、これまでのベクトルデータベース主体のRAG（Retrieval-Augmented Generation）を一歩進め、「知識のネットワーク化」による高度なエージェント記憶を提供するフレームワークです。今後の自律型AIの進化において、このような専用のメモリ層は不可欠なコンポーネントとなるでしょう。
