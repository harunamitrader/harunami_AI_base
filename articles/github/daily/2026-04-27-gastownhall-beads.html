これは何か

AIエージェント（Claude Code、Cursor、Windsurf等）が複雑なソフトウェア開発を自律的に進めるための「タスクベースのメモリ・システム」です。標準的なMarkdownによるTODO管理の限界（コンテキストの忘却、更新漏れ、構造の欠如）を克服し、エージェントがプロジェクトの全体像を常に把握したまま長期間作業を継続できる環境を提供します。

何ができるか

エージェントは専用のCLIツール（`bd`）を介して、タスクの作成、依存関係の定義、作業の開始・完了を自律的に管理できます。Gitと密接に連携しており、タスクの進捗状況はリポジトリ内のJSONLファイルとして保存されます。これにより、ブランチを切り替えてもエージェントの「記憶」が失われず、マルチエージェントやチーム開発でもコンフリクトを最小限に抑えながら協調作業が可能です。

目立つポイント

単なるTODOリストではなく、「依存関係を認識するグラフ構造」であることが最大の特徴です。エージェントは「何がブロックされているか」「次に何をすべきか」を論理的に判断できます。また、セマンティックな要約機能（Semantic Compaction）を備えており、完了した古いタスクを要約してコンテキスト窓をきれいに保つことで、AIの推論品質を維持します。

セットアップや使い方の流れ

npm経由でインストール（`npm install -g @gastownhall/beads`）した後、リポジトリ内で `bd init` を実行します。エージェントに対して「bdツールを使ってタスクを管理して」と指示するだけで、エージェントは `bd list` で未着手タスクを確認し、`bd claim` でタスクを開始する自律的なワークフローを開始します。

どんな人向けか

AIエージェントに「1分で終わる修正」ではなく、「数時間から数日かかる新機能の実装」を任せたい開発者に最適です。特に、エージェントが途中で何をしていたか忘れてしまったり、不整合なTODOを残して混乱したりすることに悩んでいるプロジェクトにおすすめです。

注意点

タスクの詳細はリポジトリ内の `.beads/` ディレクトリに保存されるため、これらをGitで管理する必要があります。また、エージェントにこのツールの使い方（bdコマンドの仕様）を理解させるためのプロンプトや、MCP経由での連携が推奨されます。

まとめ

Beadsは、AIエージェントを「その場限りのアシスタント」から「長期的なタスクを完遂する頼れる同僚」へと進化させるインフラです。Gitバックエンドによる堅牢な永続化と、グラフ構造による論理的なタスク管理は、自律型開発エージェントの次なるスタンダードになる可能性を秘めています。
