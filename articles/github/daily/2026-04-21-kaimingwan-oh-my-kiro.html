これは何か

oh-my-kiro (OMK) は、AI コーディングエージェントに「永続的なメモリ」「決定論的なワークフロー」「自己進化する知能」を付与するためのフレームワークです。シェルのカスタマイズで定番の「oh-my-zsh」の哲学をエージェントの世界に持ち込み、プロンプトベースの曖昧な指示ではなく、コード（フック）による厳格な挙動制御と長期的な記憶保持を可能にすることを目指しています。

何ができるか

エージェントの行動の節目に「フック（Hooks）」を挿入できます。例えば、機密情報の流出を阻止する block-secrets.sh、コード書き込み後の自動処理 post-write.sh、push 前のリンター強制実行など、プロジェクト固有のルールを「As-Code」でエージェントに守らせることができます。また、「RALPH」と呼ばれる構造化された思考サイクル（Read-Analyze-Loop-Perform-Halt/Evaluate）を導入し、エージェントが場当たり的ではなく、一貫した計画に基づいて動作することを保証します。

目立つポイント

単なる「便利なプロンプト集」に留まらず、エージェントに「メタ認知」と「自律的な改善」のループを組み込もうとしている点が非常にユニークです。エージェントが過去の失敗や成功を「永続メモリ」に蓄積し、同じ状況に直面した際に前回の教訓を自ら引き出すことで、使い込むほどに賢くなる自分専用の AI パートナーを構築できます。設定がシェルスクリプトや Python で記述可能なため、カスタマイズ性が極めて高いのも特徴です。

セットアップや使い方の流れ

リポジトリを ~/.kiro などのディレクトリにクローンし、Claude Code や Codex などのエージェントの作業環境にリンクさせることで導入します。同梱されている install.sh を実行すると標準的なフックセットが配置され、即座に「管理されたエージェント環境」が整います。その後は、必要に応じて plugins/ ディレクトリに独自のロジックを追加することで、エージェントを特定のプロジェクトに最適化させていきます。

どんな人向けか

自律型エージェントの「予測不可能な挙動」や「記憶の揮発性」に悩んでいる中上級の開発者に最適です。企業やプロジェクトのコーディング規約をエージェントに強制したい管理者や、エージェントを「ただのチャット相手」ではなく「信頼できる自動化基盤」へと押し上げたいと考えているエンジニアにとって、最高のカスタマイズ基盤となるでしょう。

注意点

フックによってエージェントの行動を強く制限するため、スクリプトの設定を誤るとエージェントが「何もできない」状態になったり、無限ループに陥ったりする可能性があります。また、現時点では「エージェント版 oh-my-zsh」としての構想の初期段階にあるため、各エージェントツール（Claude Code 等）との細かな統合設定については、ユーザーがソースコードを確認しながら調整する必要があります。

まとめ

oh-my-kiro は、AI エージェントという「馬」を乗りこなすための、洗練された「手綱」と「サドル」のような存在です。プロンプトエンジニアリングの限界を、伝統的なシステム管理の知恵（フックとワークフロー）で突破しようとする挑戦的なアプローチは、実務でエージェントを運用する上で極めて現実的な解決策を示しています。エージェントを本当の意味で「使い物」にしたい方は、一考の価値があるリポジトリです。
