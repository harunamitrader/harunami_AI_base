BrowserWing

ブラウザ操作をAIエージェント向けのMCPコマンドやClaude Skillへ変換し、自律的なブラウザ自動化を実現するプラットフォーム。

記事作成日 2026-04-24
種別 Pickup
対象 repo browserwing/browserwing
Star数 1,200

これは何か

AIエージェントがブラウザを効率的かつ確実に制御できるように設計された、ブラウザ自動化プラットフォームです。手動で行ったブラウザ操作をMCP（Model Context Protocol）コマンドやClaude Skill（SKILL.md）に変換し、AIが直接ブラウザを「手」として使えるようにします。

何ができるか

78種類のプリセットスクリプト（GitHubトレンド、SNS、ニュース抽出等）を即座に実行できるほか、独自の操作をビジュアルレコーダーで記録し、AI用のスキルとして書き出すことが可能です。CLIモード、MCPサーバー、HTTP APIなど、多様なインターフェースでエージェントと統合できます。

目立つポイント

LLMのトークン消費を抑える設計が特徴で、AIに生のHTMLを読ませるのではなく、構造化されたデータや特定のコマンド実行を委ねることで、高速かつ安価な自動化を実現します。セッション管理（Cookie/認証維持）も強力で、ログインが必要なサイトの操作も安定して行えます。

セットアップや使い方の流れ

npm install -g browserwing で簡単にインストール可能です。起動後はローカルでMCPサーバーとして動作し、CursorやClaude CodeなどのAIツールに設定を貼り付けるだけで、チャット越しにブラウザ操作を指示できるようになります。

どんな人向けか

AIエージェントにWeb情報の収集やフォーム入力などの実務を任せたい開発者、トークンコストを抑えつつブラウザ自動化を実現したいエンジニア、あるいは独自のブラウザ操作スキルをAIに教え込みたいユーザーに最適です。

注意点

自動化の際は対象サイトの利用規約を遵守する必要があります。また、インストール時にChrome/Chromiumが必要となります。macOSユーザーは初回の実行許可設定（xattr）が必要な場合があるため、公式ドキュメントの指示に従ってください。

まとめ

BrowserWingは、AIエージェントにとっての「ブラウザ操作の標準インターフェース」を目指す野心的なプロジェクトです。MCPやSkillsプロトコルへのネイティブ対応により、AIがWebという巨大なリソースをより自由かつ確実に扱える未来を提示しています。
