これは何か

openclaude は、Claude Code 系の CLI を Anthropic 固定ではなく、OpenAI 互換 API を話せる任意のモデルで動かすための fork です。README では GPT-4o、Gemini、DeepSeek、Llama、Mistral、Ollama など幅広い接続先を想定し、CLI コマンド名はそのまま openclaude として配布しています。

何ができるか

このリポジトリの売りは、元のツール体験を大きく崩さずにプロバイダだけ差し替える点です。README 上では bash、ファイル read/write/edit、grep、glob、agents、tasks、MCP、画像入力、スラッシュコマンド、サブエージェント、永続メモリまで動くと整理されています。

目立つポイント

中心は src/services/api/openaiShim.ts で、Anthropic 形式のメッセージや tool_use / tool_result、ストリーミングイベントを OpenAI Chat Completions API 向けに変換します。README では追加差分を 6 ファイル・依存追加なしとしており、fork の狙いを「全面改造」ではなく「接続層の差し替え」に絞っているのが分かりやすいです。

セットアップや使い方の流れ

最短は npm install -g @gitlawb/openclaude です。起動前に CLAUDE_CODE_USE_OPENAI=1、OPENAI_API_KEY、OPENAI_MODEL を設定し、必要なら OPENAI_BASE_URL で接続先を差し替えます。README には bun run smoke、doctor:runtime、hardening:check などの診断系コマンドもまとまっていて、導入後の事故を減らしやすい構成です。

どんな人向けか

Claude Code 風の CLI ワークフローを、OpenAI 系やローカル LLM でも試したい人に向いています。特に、Ollama や LM Studio のようなローカル環境を含めて、同じ操作感で複数モデルを比較したい人には分かりやすい選択肢です。

注意点

README 自体が、出発点として 2026 年 3 月 31 日の source map 露出を明記しており、教育・研究用途の位置づけも強く打ち出しています。また、extended thinking や prompt caching のような Anthropic 固有機能はそのままでは再現されません。実運用前提なら、法的背景と「何が同じで何が違うか」を README の Disclaimer と Difference 節で確認した方がいいです。

まとめ

openclaude は、Claude Code 系の操作感を維持したまま、モデル選択の自由度だけを大きく広げたい人向けの fork です。価値の中心は多機能さよりも「既存の CLI 体験を OpenAI 互換 API に移植する shim 設計」にあるので、マルチモデル化の実装例として読むと収穫があります。
