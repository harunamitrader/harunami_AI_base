これは何か

AIコーディングエージェント（Claude Code, Codex, Cursor, Copilot, Windsurf等）を、単なるチャットUIから「仕様に基づいて自律的にタスクを完遂するSDLC（ソフトウェア開発ライフサイクル）エンジン」へと変貌させるためのSDD（仕様駆動開発）ハーネスです。仕様書を「AIと人間の契約」として扱い、承認された仕様から自律的な実装を長時間駆動させることができます。

何ができるか

プロジェクトの初期構想から、要件定義（EARS形式）、アーキテクチャ設計、具体的なタスク分割までのフローをAIがサポートします。そして最大の目玉は /kiro-impl コマンドによる自律実装です。定義された tasks.md に基づき、AIがタスクごとに実装、テスト実行（TDD）、独立したレビュアーによる検証、そして失敗時の自動デバッグ（Auto-debug）を繰り返し、人間が介入することなく機能を完成させます。

目立つポイント

単一のプロンプトで実装させるのではなく、実装者（Implementer）とレビュアー（Reviewer）という役割を分離し、さらにエラー時には別の文脈で原因を調査する「Auto-debug」機能を備えている点が極めて強力です。これにより、既存のAIエージェントが陥りがちな「一度詰まるとループする」問題を克服し、複数のファイルを横断する大規模な機能追加でも高い成功率を維持します。また、主要な8種類のコーディングエージェントに同一の「Agent Skills」セットを提供できる互換性も備えています。

セットアップや使い方の流れ

プロジェクトのルートディレクトリで npx cc-sdd@latest を実行するだけで導入可能です。その後、各エージェント（例：Claude Code）を起動し、/kiro-discovery <作りたい機能> を入力することで、仕様の策定から実装までのガイドが開始されます。生成された仕様書（requirements.md, design.md, tasks.md）を人間がレビュー・承認すれば、あとは /kiro-impl で自律実装を開始するだけです。

どんな人向けか

AIを使って、より大規模で複雑な機能を「指示一発」に近い感覚で実装したい開発者。あるいは、AIに任せきりにするのではなく、仕様の段階で境界線（Boundary）を明確にし、設計品質を担保しながら自動化の恩恵を受けたいチームに最適です。

注意点

v3.0へのメジャーアップデートにより、従来のコマンド体系から Agent Skills ベースのワークフローに刷新されています。また、実装は TDD（テスト駆動開発）を前提としているため、プロジェクトに適切なテスト環境が整っていることが望ましいです。

まとめ

cc-sddは、AIによる開発を「その場しのぎのパッチ当て」から、エンジニアリングの原則に基づいた「構造化された自律実行」へと引き上げるためのツールです。仕様書という古典的かつ本質的な資産を、AIエージェントを動かすための最高級の燃料へと変えてくれます。
