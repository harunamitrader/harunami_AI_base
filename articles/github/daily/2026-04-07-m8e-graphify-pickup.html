記事作成日 2026-04-07
種別 Pickup
対象 repo m8e/graphify

これは何か

graphify は、ローカルフォルダ内のソースコード、論文、画像、エンジニアリングノートなどを自動的にスキャンし、それらの相互関連性を可視化する「ナレッジグラフ」を構築するための Claude Code スキルです。 雑多な情報を構造化することで、AI エージェントがプロジェクトの全体像を把握する際の効率を極限まで高めます。

何ができるか

  • 自動構造化と可視化: フォルダ内のファイル間の関係性を抽出し、インタラクティブな HTML グラフを生成。
  • 多彩なエクスポート: Obsidian vault 形式や Wiki 形式での出力に対応し、既存のナレッジツールと連携可能。
  • グラフ・レポート生成: 主要な接続や重要な問いを抽出した GRAPH_REPORT.md を自動作成。
  • トークン消費の劇的な削減: 生のテキストファイルをすべて読み込ませるのに比べ、グラフ構造を活用することでコンテキスト消費を大幅に（最大約70倍）節約。

目立つポイント

最大の強みは、単純な全文検索（RAG）を超えた **「文脈の構造的理解」** を AI にもたらす点です。 数週間後にプロジェクトを再開した際でも、構築済みのグラフを読み込むだけで、AI エージェントは即座に複雑な依存関係や背景知識を思い出すことができます。 SHA256 によるキャッシュ機能も備えており、変更があったファイルのみを効率的に再処理します。

セットアップや使い方の流れ

  1. Claude Code がインストールされている環境で、このスキルをディレクトリに導入します。
  2. 「このディレクトリを graphify して」と指示するだけで、スキャンとグラフ構築が始まります。
  3. 出力された HTML をブラウザで開けば、動的なグラフで情報の繋がりを確認できます。
  4. GRAPH_REPORT.md を通じて、プロジェクトの要諦を AI に把握させることができます。

どんな人向けか

  • 大規模、あるいは長期間にわたる複雑なプロジェクトを AI エージェント（Claude Code 等）と共に進めている開発者。
  • Obsidian などのツールを使っており、ローカルの資料をナレッジグラフとして整理・活用したい人。
  • LLM のコンテキスト制限やトークンコストを抑えつつ、深い文脈理解を AI に期待したい人。

注意点

大量のファイルを一度に処理する場合、初回のグラフ構築には相応の時間がかかることがあります。また、出力されるグラフの精度はソースとなるドキュメントやコードの記述密度に依存します。

まとめ

graphify は、AI エージェントにとっての「外部脳」を、構造化されたグラフという形で提供する強力なツールです。 情報の「点」を「線」で繋ぎ、AI がプロジェクトを文字通り「俯瞰」できるようにすることで、次の次元の AI 駆動型開発を支えます。
