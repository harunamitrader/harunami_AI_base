これは何か

MANO-P は、MININGLAMP-AI チームによって開発された、GUI（グラフィカル・ユーザー・インターフェース）の操作を自律的に行うための視覚言語アクション（VLA）エージェントです。 OSWorld ベンチマークにおいて、専門的な GUI エージェントモデルの中でトップとなる成功率 58.2% を記録し、Gemini 2.5 Pro や Claude 4.5 といった名だたる LLM のコンピュータ操作能力をも上回る性能を示しています。 最大の特徴は、API を必要とせず、人間と同じように「画面を見て判断し、マウスとキーボードで操作する」という点にあります。

何ができるか

  • クロスアプリの自動化: API の提供されていない古い基幹システムやデスクトップアプリ間でのデータ移行・操作を自動化。
  • 複雑な多段階タスクの実行: 「Think-Act-Verify（思考・実行・検証）」のループにより、100 ステップを超えるような長いワークフローを自律的に完遂。
  • エッジデバイスでのローカル推論: クラウドにデータを送ることなく、Mac mini や MacBook Pro などのローカル環境で推論と実行が可能。
  • システムの自動巡回と検証: 実行したアクションが期待通りになったかを自らスクリーンショットで確認し、エラーがあれば自己修正。

目立つポイント

これまで GUI エージェントは「計算負荷が高い」「精度が低い」という課題がありましたが、MANO-P は「混合精度量子化」や「視覚トークンの削減」といった独自技術により、MacBook のようなエッジデバイスでも実用的な速度で動作するように最適化されています。 教師あり学習（SFT）の後にオフライン・オンライン両方の強化学習を組み合わせる 3 段階の学習プロセス（Mano-Action 手法）により、GUI 特有の曖昧な配置や動的な変化への対応力が格段に向上しています。

セットアップや使い方の流れ

Python 環境が構築された Mac や Linux 環境で利用可能です。リポジトリをクローンした後、事前に学習済みのモデルウェイト（Hugging Face 等から配布）をロードして使用します。 推論エンドポイントを立ち上げ、VNC 経由や OS 本体のアクセシビリティ権限を通じてデスクトップへの制御権を与えることで、エージェントが自律的にマニピュレーションを開始します。

どんな人向けか

API が開放されていないレガシーシステムの自動化に悩む DX 担当者や、プライバシーの観点からクラウド AI に画面を見せることができない環境での作業自動化を検討している組織に最適です。 また、OS 操作エージェントの最先端技術を研究したいエンジニアにとっても、ベンチマーク 1 位のアルゴリズムは非常に興味深い対象となります。

注意点

エッジデバイス向けに最適化されているとはいえ、スムーズな推論には Apple Silicon（M1/M2/M3 等）の GPU や十分なメモリ容量が推奨されます。 また、操作権限を AI に与えるため、予期せぬブラウザの閉じや重要データの削除を防ぐため、サンドボックス環境や仮想マシン（VM）上での実行が強く推奨されています。

まとめ

MININGLAMP-AI/MANO-P は、AI が「汎用的な道具」としてコンピュータを使いこなす未来を強力に引き寄せるリポジトリです。 OS レベルの操作を API なしで高精度にこなす能力は、あらゆる単純作業を AI に任せるための基盤技術となる可能性を秘めています。
