これは何か

Claude Code や OpenClaw などの AI コーディングエージェントが消費する「トークン」を徹底的に監視・最適化するためのツールです。 多くのツールがターミナル出力の圧縮のみに焦点を当てる中、このプロジェクトはシステムプロンプトの重複、肥大化した設定、未使用のスキル、そして自動要約（compaction）による情報喪失など、トークン消費のあらゆる「死角」をカバーします。

何ができるか

単なる圧縮ツールを超えた、総合的なコンテキスト管理機能を提供します。

  • トークン可視化ダッシュボード: ローカルマシン（localhost:24842）で動作するダッシュボードにより、ターンごとのコスト、キャッシュヒット率、コンテキスト品質の推移をリアルタイムで確認できます。
  • Smart Compaction: Claude Code の自動要約が走る直前に重要な状態をチェックポイントとして保存。要約によって「AI が忘れてしまった」重要なコンテキストを自動的に復元します。
  • Delta Mode: ファイルを編集した後に再度読み込む際、全文ではなく「差分（diff）」のみを AI に送ることで、読み取りトークンを 90% 以上削減します。
  • ループ検知: AI が同じエラーでリトライを繰り返している状況を検出し、インラインで通知することで無駄な消費を食い止めます。

目立つポイント

徹底した「ローカルファースト・低オーバーヘッド」設計が魅力です。 外部へのテレメトリ送信は一切なく、データはローカルの SQLite に保存されます。さらに、このツール自体が AI のコンテキストウィンドウ（1M トークンなど）を一切消費しないよう外部プロセスとして動作するため、貴重な作業枠を圧迫しません。 ヘビーユーザーであれば、キャッシュ効率の向上と無駄なリトライの防止により、月に数百ドル規模のコスト削減が見込めるとしています。

セットアップや使い方の流れ

  1. Claude Code 内でプラグインとしてインストールするか、リポジトリをクローンして install.sh を実行します。
  2. python3 measure.py setup-daemon を実行してバックグラウンド監視を開始します。
  3. ブラウザで http://localhost:24842/token-optimizer を開き、ダッシュボードを表示します。
  4. あとは普段通り AI エージェント（Claude Code 等）を使用するだけで、バックグラウンドで最適化と計測が行われます。

どんな人向けか

  • Claude Code や OpenClaw を業務で長時間利用しており、API コストを最適化したい開発者。
  • 長時間におよぶデバッグや数千行のプロジェクトを扱っており、コンテキストが満杯になった際の「AI の劣化」を肌で感じている人。
  • AI の思考プロセスやコンテキストの中身、キャッシュの効き具合を詳細に分析したい技術者。

注意点

Delta Mode などの一部の強力な機能は、AI がコンテキストを正しく把握できなくなるリスクを避けるため、一定のしきい値で安全策（全文読み込みへのフォールバック）が働くようになっています。 また、現時点では Claude Code と OpenClaw が主なターゲットであり、Cursor や Windsurf はロードマップ上の対応待ちとなっています。

まとめ

token-optimizer は、AI エージェントを「より賢く、より安く、より長く」使い続けるための必須級サポーターです。 「コンテキストが 1M に増えたから安心」ではなく、増えたからこそ発生する「品質低下」と「コスト増」を科学的に解決してくれる、実務志向の強いツールと言えます。
