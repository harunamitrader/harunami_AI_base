これは何か

indexion は、大規模かつ複雑なソースコードの構造を可視化し、分析・検索・ドキュメント生成を劇的に効率化するためのツールキットです。 「地図は領土ではない (The map is not the territory)」という設計思想のもと、コードの抽象的な構造を KGF (Kindred Graph Format) という独自形式で扱い、人間と AI の両方がプロジェクトを深く理解するための「動点な地図」を提供します。

最近注目を集めている MoonBit 言語で開発されており、高速な動作と先進的な機能を両立させているのが特徴です。

何ができるか

  • リファクタリングの自動計画: コード間の類似度を分析し、重複の統合や不要なラッパーの削除（unwrap）など、具体的な改善プランを提示します。
  • セマンティック & 構造的検索: 自然言語による検索に加え、言語の構造（Tree-sitter活用）を理解した Structural Grep が可能です。
  • MCP (Model Context Protocol) サーバー: Claude Code や Cursor などの AI エージェントと連携し、AI に対してプロジェクト全体の正確な文脈を提供します。
  • インテリジェント・ドキュメント生成: コードの構造解析に基づき、README、Wiki、API リファレンスを自動生成します。

目立つポイント

単にコードを検索するだけでなく、「次に行うべきアクション」を提示する plan コマンド が非常に実用的です。 また、MCP サーバーとして機能するため、AI エージェントが「迷子」になることを防ぎ、トークンコストを抑えながら確実なコード操作を支援します。

セットアップや使い方の流れ

ツールは indexion コマンドとしての CLI 提供が主軸です。 例えば、プロジェクトの類似性を調査するには indexion explore を実行し、改善が必要な箇所を特定した後は indexion plan refactor で具体的な手順を生成します。 AI と連携させる場合は、MCP 構成に indexion mcp を追加するだけで、エージェントが indexion の解析能力をフル活用できるようになります。

どんな人向けか

  • 大規模プロジェクトのメンテナー: 技術負債の解消や、複雑化した構造の整理を自動化したい方。
  • AI 駆動開発を極めたい開発者: Claude Code や Cursor 等の AI エージェントに、より正確な「プロジェクトの地図」を与えたい方。
  • ドキュメント管理を自動化したいチーム: コードとドキュメントの乖離を防ぎ、常に最新の情報を維持したい方。

注意点

現在活発に開発されているプロジェクト（プレリリース段階）であるため、機能の追加や変更が頻繁に行われる可能性があります。 最新の機能を利用するには、ソースからのビルドや定期的なアップデートが推奨されます。

まとめ

indexion は、単なる静的解析ツールを超え、AI 時代のソフトウェア開発における「コンテキスト（文脈）のインフラ」を目指す野心的なプロジェクトです。 コードを「読む」ストレスを軽減し、人間と AI が協力してより良い設計を目指すための強力な味方となるでしょう。
