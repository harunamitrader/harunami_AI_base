これは何か

X (Twitter) の機能をターミナルから直接利用するための、強力なコマンドライン・インターフェース (CLI) です。ブラウザのセッションクッキーを使用することで、煩雑な公式 API キーの取得や申請を行うことなく、エンジニアにとって馴染み深い CUI 上で全ての Twitter 操作を完結させることができます。

何ができるか

このツール一つで、普段ブラウザやアプリで行っているほぼ全ての操作が可能です。

  • 閲覧と検索: タイムラインの表示、ブックマークの取得、キーワード検索、特定ユーザーの投稿一覧の参照。
  • 投稿とインタラクション: 新規ツイート（返信含む）、リツイート、いいね、ブックマークの追加・削除、ツイートの削除。
  • データの構造化出力: 取得したツイートやユーザー情報を YAML または JSON 形式で出力可能。AI エージェントにデータを渡したり、スクリプトで自動処理したりする際のデザイン性が非常に高いです。
  • セキュリティと検知回避: TLS フィンガープリントの偽装や、リクエスト間隔のゆらぎ（jitter）機能などを備え、自動操作として検知されにくい工夫が施されています。

目立つポイント

「API キーが不要」という手軽さはもちろん、何より「エンジニアのワークフローに Twitter を統合できる」点が秀逸です。特に情報のエクスポート機能が充実しているため、例えば「自分のブックマークを AI に読み込ませて整理させる」「タイムラインの特定のトピックを自動で要約する」といった、AI と X を組み合わせた自律型エージェントのナレッジソースとして極めて優秀なパーツになります。

セットアップや使い方の流れ

  1. uv や pipx 等を使って twitter-cli をインストールします。
  2. 普段使いのブラウザ（Chrome, Edge 等）で X にログインした状態で、ツール内のコマンド（twitter-cli auth 等）を実行してクッキーを紐付けます。
  3. twitter-cli home でタイムラインを見たり、twitter-cli post "Hello world" で投稿したりします。
  4. AI エージェントに操作させたい場合は、MCP ツール等を通じてこの CLI を呼び出すように構成します。

どんな人向けか

  • Twitter を情報のインプット源としてバリバリ使いつつ、アウトプットもコマンド一つで済ませたい効率化オタクのエンジニア。
  • AI エージェントに「自分の関心事（ブックマーク）」を学習させたいが、API の制限や申請に疲弊している人。
  • ブラウザを開かずに X の世界に浸りたい、ターミナル中毒の住人。

注意点

公式 API を介さないツールであるため、過度なリクエストはアカウントに制限がかかるリスクがあります。ツールの推奨設定（ディレイ機能等）に従い、常識的な範囲での利用が求められます。

まとめ

twitter-cli は、ソーシャルメディアという広大なデータを「開発者の手元」へと引き込み、自在に加工・活用することを可能にします。AI との親和性も高いこのツールは、単なるクライアントソフトを超えた、知識マネジメントの強力なフロントエンドとなるでしょう。
