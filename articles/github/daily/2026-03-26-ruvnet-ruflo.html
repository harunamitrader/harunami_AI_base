これは何か

rufno は、Anthropic の Claude Code を単なる一人の「AI アシスタント」から、高度に調整された「マルチエージェント・チーム」へと進化させるためのオープンソース・オーケストレーション基盤です。以前は Claude Flow として知られていました。

何ができるか

60 種類を超える専門エージェント（コーダー、テスター、セキュリティ監査人、ドキュメント作成者など）を「Hive Mind」と呼ばれる集合知モデルで統合し、複雑なソフトウェア開発タスクを協調して実行できます。Claude Code 内で直接 MCP (Model Context Protocol) サーバーとして動作し、コマンド一つでスウォーム（群れ）を起動して並列・段階的なタスク処理が可能です。

目立つポイント

最大の特徴は、Strategic/Tactical/Adaptive といった役割を持つ「Queen」エージェントが、他の Worker エージェントを指揮して合意（コンセンサス）を形成するガバナンス構造です。また、自己学習アルゴリズムを搭載しており、成功したワークフローのパターンを学習して時間とともに最適化される点もユニークです。

セットアップや使い方の流れ

Node.js 環境があれば npx ruflo@latest init で初期化できます。Claude Code と連携させるには npx ruflo@latest mcp start で MCP サーバーを起動し、Claude 側で特定のタスク（例：「認証機能を実装し、テストを書いてドキュメントを作成せよ」）を投げると、適切なエージェント群が自動的に選定・編成されます。

どんな人向けか

Claude Code を既に業務に取り入れており、単一のプロンプトでは対応しきれない大規模なリファクタリングや、包括的なセキュリティ監査、自動テストスイートの構築を AI に自律的に行わせたい開発者に最適です。

注意点

60 種類以上のエージェントを動かす「スウォーム」構成の場合、API コストが急増する可能性があります。Ruflo はコスト効率の高いルーティング機能も備えていますが、並列実行数や複雑な合意形成アルゴリズムの使用には注意が必要です。

まとめ

Ruflo は、Claude Code をエンタープライズレベルの「自律型開発チーム」へ拡張する、現在最も注目されるエージェント・オーケストレーターの一つです。個別のツールとしてではなく、既存の Claude ワークフローを深める「基盤」として設計されている点が強力です。
