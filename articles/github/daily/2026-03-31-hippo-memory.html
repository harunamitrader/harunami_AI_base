これは何か

hippo-memoryは、AIエージェント（Claude Code、Cursor、OpenClawなど）に「生物学的な海馬」の機能を付与するオープンソースのメモリシステムです。従来の単純なデータ保存とは異なり、情報の減衰（忘却）、検索による強化（定着）、そして「睡眠」を通じた記憶の統合といった脳の仕組みを模倣しています。エージェントが過去の経験やエラーから自律的に学習し、同じミスを繰り返さない「成長するパートナー」へと進化させることを目的として設計されています。

何ができるか

このシステムは、hippo rememberによる記憶の保存とhippo recallによる関連情報の呼び出しを基本としつつ、高度な自律学習機能を備えています。hippo learn --gitコマンドを使用すれば、コミット履歴から自動的に教訓や知見を抽出可能です。また、毎日午前6:15に行われる自動タスク（cron）によって記憶の整理が行われ、重要な情報は強化され、不要な情報は自然に「忘却」されることで、常に最適なコンテキストを維持します。さらに、hippo shareを使えば異なるプロジェクト間での知見共有も可能です。

目立つポイント

最大の特徴は、Node.js 22.5以上の組み込み機能のみで動作する「実行時依存関係ゼロ」の軽量設計です。外部データベースを必要とせず、SQLiteと人間が読めるMarkdown/YAML形式の両方で記憶を管理するため、Gitでの履歴管理も容易です。また、hippo dashboardによる記憶状態の可視化や、ベンチマークにおけるエージェントのミス率（78%から14%へ）の劇的な改善など、実用性と透明性の両立が非常に高いレベルで実現されています。

セットアップや使い方の流れ

導入は非常にシンプルで、npm install -g hippo-memoryでインストールした後、プロジェクトディレクトリでhippo initを実行するだけです。これによりCLAUDE.mdや.cursorrulesが自動検出され、エージェントが自律的にメモリを参照できるようパッチが適用されます。セットアップ後は、エージェントが作業中に自動で記憶を読み書きするほか、手動での記録や、ブラウザでのダッシュボード確認などを通じて、記憶の定着具合をコントロールできます。

どんな人向けか

Claude CodeやCursorなどのAIエージェントをヘビーに使いこなしており、エージェントに同じ説明を何度も繰り返す手間を省きたい開発者に最適です。また、複数のプロジェクトを並行して進めており、あるプロジェクトで得た知見を別のプロジェクトのAIにも即座に反映させたいマルチプロジェクト開発者にとっても、非常に強力な武器となります。システムプロンプトの肥大化に悩んでいるユーザーにも推奨されます。

注意点

Node.js 22.5以降の環境が必須となるため、古いランタイム環境では動作しません。また、「忘却」という生物学的な特性を模倣しているため、重要ではないと判断された情報は時間とともに消える可能性があります。絶対に忘れてほしくない重要な制約事項などは、pin機能を使用して保護する運用が必要です。高度なハイブリッド検索を利用するには別途@xenova/transformersのインストールが推奨されます。

まとめ

hippo-memoryは、AIエージェントの記憶を「詰め込み」から「自律的な学習プロセス」へと変える画期的なツールです。開発者の活動（コミットなど）から静かに教訓を積み重ねていくこの仕組みは、AIを単なるツールから真の「同僚」へと昇華させる可能性を秘めています。依存関係の少なさと高い統合性により、今後のエージェント開発における標準的なメモリ基盤となることが期待されます。
