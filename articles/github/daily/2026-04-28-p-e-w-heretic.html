これは何か

「Heretic」は、Transformerベースの言語モデルに施された検閲（安全アライメント）を、追加学習なしで自動的に除去するためのツールです。最新の「アブリタレーション（Abliteration）」技術を駆使し、モデル内部の「拒絶方向」を特定して中和することで、制限のない自由な回答を可能にします。

何ができるか

専門的な知識がなくても、コマンド一つでモデルの「脱獄」が可能です。具体的には、拒絶反応の最小化と、元のモデルからのKLダイバージェンス（モデルの変質度合い）の最小化を同時に最適化することで、知能や推論能力を損なうことなく検閲のみを効率的に取り除いたモデルを生成できます。

目立つポイント

最大の特徴は、Optunaを用いたパラメータ最適化（TPE）を導入している点です。これにより、これまで人間が手動で行っていた繊細な調整作業を完全に自動化し、専門家が作成したモデルと同等、あるいはそれ以上の品質の非検閲モデルを生成できます。また、研究向け機能として、モデル内部の隠れ状態を可視化する機能も備えています。

セットアップや使い方の流れ

Python 3.10+ 環境で `pip install heretic-llm` を実行してインストールします。その後、`heretic --model <モデル名>` コマンドを叩くだけで最適化プロセスが開始されます。RTX 3090クラスの環境であれば、Llama-3.1-8B-Instruct の処理に約45分ほどかかります。処理後はそのままチャットでテストしたり、Hugging Faceにアップロードしたりすることが可能です。

どんな人向けか

AIの研究者、ローカルLLM愛好家、あるいは「モデルが特定のトピックに対して回答を拒否すること」にフラストレーションを感じている開発者に最適です。特に、GemmaやQwenなどの強力なモデルを、本来の性能のまま自由に使いたい場合に非常に強力な武器となります。

注意点

検閲を解除したモデルは、有害な情報や不適切なコンテンツを生成する可能性があるため、使用には倫理的・法的な責任が伴います。また、強力なモデルの処理には相応のVRAM（bitsandbytesによる4bit量子化オプションもあります）が必要になる点に注意してください。

まとめ

Hereticは、ブラックボックス化されがちな「安全アライメント」という壁を、数学的・アルゴリズム的に打破しようとする野心的なプロジェクトです。自動化された高品質なアブリタレーションは、今後のモデルカスタマイズの標準になるかもしれません。
