これは何か

ccremote は、Claude Code セッションを tmux 上で監視し、Discord から承認や状態確認をできるようにする CLI です。README では Remote approvals、quota-aware auto-continuation、quota scheduling を主軸にしており、席を離れた間に止まりやすい Claude Code の運用を補います。

何ができるか

Discord からファイル編集や shell 実行の承認を返せるほか、クォータ制限で止まったセッションを検知して 5 時間ウィンドウ後に自動再開できます。さらに完了通知、/output による直近出力の確認、セッション一覧や停止、毎日のダミー実行によるクォータ時間帯の調整まで用意されています。

目立つポイント

単なる通知ボットではなく、ccremote 自体を Claude Code 起動コマンドの置き換えとして使える点が分かりやすいです。ccremote だけで start に入り、tmux セッション生成、5 秒の情報表示、Claude Code への attach、バックグラウンド監視をひと続きで処理します。Discord 側もセッションごとの private channel を作る前提で、遠隔観察と承認が分離されにくい設計です。

セットアップや使い方の流れ

導入は Node.js 20.19.4 以上、tmux、Discord bot を用意し、npm install -g ccremote のあと ccremote init で設定を作る流れです。普段は ccremote または ccremote start --name \"my-session\" で監視付きセッションを起動し、必要に応じて ccremote schedule、ccremote resume、ccremote status を組み合わせます。

どんな人向けか

Claude Code を長時間走らせる人、承認ダイアログで止まるのが困る人、夜間や外出中も状態を確認したい人に向いています。特に複数セッションを並行で回す運用では、Discord 通知と session list がかなり実務的です。

注意点

前提に Discord bot 構築と tmux 環境があり、README でも macOS 付属 tmux の既知不具合に触れています。加えて、承認や出力の一部を Discord に流す以上、bot の権限、private channel 設定、プロジェクトごとの bot 分離など運用面を整理して使う必要があります。

まとめ

ccremote は、Claude Code の「その場にいないと止まる」問題をかなり直接的に潰すツールです。ローカル CLI 体験を保ったまま、遠隔承認、完了通知、クォータ再開を足したい人には実用度が高いリポジトリです。
