これは何か

Craft Agents は、craft.do 社が自社の AI エージェント業務効率化のために開発したツールをオープンソース化したものです。 Claude Agent SDK と Pi SDK を組み合わせ、CLI 操作を不要とする「ドキュメント中心（vs コード中心）」のワークフローで AI エージェントとの協働を実現します。マルチタスクの分担、外部 API への接続、 セッション共有などをプロンプトのみで制御できる Agent Native な設計が特徴です。

何ができるか

  • マルチタスク並列処理： 複数のエージェントタスクを直感的に分割・管理できる UI。
  • API・サービス接続： 煩雑さなく外部 API やサービスへ即座に接続可能。
  • セッション共有： チーム内でエージェントの作業状況を共有できる協働機能。
  • 高いカスタマイズ性： コードエディタ不要で、Craft Agents 自体を Craft Agents で改造可能。
  • Apache 2.0 ライセンス： 商用利用を含む自由なカスタマイズが可能。

目立つポイント

Claude Agent SDK をコアに採用し、Pi SDK を補完的に活用した構成です。 ドキュメントベースのワークフロー管理層、API コネクター、セッション管理、 およびビジュアル UI コンポーネントで構成されています。 開発チーム自身が Craft Agents のみを用いてプロダクトを作り続けているという実績が、 実用性の高さを示しています。

セットアップや使い方の流れ

動作確認には GitHub 上の紹介動画（YouTube リンク）が最も分かりやすいとされています。 基本的には Claude API キーを設定後、GUI ベースでタスクとエージェントを定義する形式です。 コード記述を最小限に抑えながら、AI エージェントを使ったアプリケーション開発を始められます。

どんな人向けか

非エンジニアでも AI エージェントを活用したい方、CLI 操作が苦手なビジネスユーザー、 および Anthropic の Claude をフル活用したいチームに向いています。

注意点

OSS 公開直後のプロジェクトのため、ドキュメントや API の安定性が今後変化する可能性があります。 また、Claude エージェント SDK への依存度が高いため、Anthropic の API 変更がそのまま影響します。

まとめ

Craft Agents は、AI エージェントとの協働を「コードを書く作業」から「思考と指示を与える作業」へと昇華させるツールです。 Claude Code のような CLI に慣れていないチームに対して、エージェント活用の入り口を大幅に下げる可能性を持っています。
