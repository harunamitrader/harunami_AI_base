CLI Prompt Cron

AI への定期命令を自動化。Claude Code や Gemini CLI をスケジュール実行し、ダッシュボードで監視する管理デーモン。

これは何か

Claude Code、Gemini CLI、Codex といった「対話型 AI CLI」を、決まった時間に自動実行するためのスケジューラー兼モニタリングデーモンです。「毎日○時に最新ニュースを AI に要約させる」「毎週○曜日にコードベースの健康診査を行う」といった、定型的な AI 操作を自動化するために設計されています。Node.js で動作し、実行状況をブラウザから一目で確認できる専用ダッシュボードを備えています。

何ができるか

  • 柔軟なスケジュール設定： 標準的な cron 形式を用いて、分単位での精緻な実行タイミングをジョブごとに設定できます。
  • 多彩な CLI への対応： 特定の AI ツールに依存せず、実行時に渡すコマンドライン引数を自由に設定可能。シェルスクリプト経由での複雑なパイプ処理も自動化できます。
  • ウェブダッシュボード： 稼働中のジョブ一覧、次回の実行予定、過去の成功・失敗履歴を直感的な UI で管理・監視できます。
  • 詳細なログ管理： 各ジョブの標準出力（stdout）と標準エラー出力（stderr）をファイルに保存し、ウェブ画面から過去の実行時の「AI の回答」を遡って確認可能です。
  • 環境変数の制御： 作業ディレクトリや必要な環境変数を、個別のジョブ定義の中にカプセル化して管理できます。

目立つポイント

AI エージェントが実用段階に入りつつある今、最大の課題は「毎回のプロンプト投入」を自動化することです。CLI Prompt Cron は、これまで「手動でコマンドを打つ」必要があった AI 連携を「バックグラウンドで勝手に動くプロセス」へと格上げしてくれます。特に、ブラウザベースの監視画面によって「今朝の自動リサーチは成功したか」をスマホや PC から即座に確認できる点は、運用の安心感を大きく高めています。

セットアップや使い方の流れ

  1. リポジトリをクローンし、npm install でパッケージを導入します。
  2. data/jobs/ ディレクトリに、JSON 形式でジョブ定義ファイルを作成します（cron 式、実行コマンド等を記述）。
  3. npm start でサーバーを起動します。
  4. 設定したポート（デフォルト 3000 番など）にブラウザでアクセスし、ダッシュボードを表示します。
  5. 時間になると自動で AI CLI が呼び出され、処理結果がログに蓄積されます。

どんな人向けか

  • GitHub のトレンド調査や株価分析など、AI を使った定型リサーチをルーチン化したい開発者。
  • CI/CD 以外のパイプラインで、AI による定期的なコードレビューやドキュメント自動生成を行いたいチーム。
  • 複数の AI CLI ツールを使い分けており、その実行履歴を一箇所で集中的に管理したいユーザー。

注意点

AI CLI ツールの多くはインタラクティブ（対話型）な動作を想定しているため、自動実行時には「対話の強制終了（--yes フラグなど）」の設定が必要です。また、API 料金が発生するツールをスケジュール実行する場合は、予期せぬ実行回数にならないよう cron 設定に注意を払う必要があります。

まとめ

CLI Prompt Cron は、AI の「能動的な活用」を支える重要なミドルウェアです。私たちが AI の前に座っていない時間にも、指定されたプロンプトが忠実に実行され、知見がログとして積み上がっていく。そんな「AI が自走する環境」を構築するための、シンプルながらも強力な基盤と言えるでしょう。
