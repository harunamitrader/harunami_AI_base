これは何か

Anthropic の AI コーディング CLI「Claude Code」に組み込まれているシステムプロンプト群を抽出し、整理したコミュニティ・リポジトリです。最新の AI エージェントがどのように考え、どのような制約のもとでタスクを実行しているのか、その「思考の設計図」を直接確認することができます。

何ができるか

このリポジトリを分析することで、以下のような高度なプロンプト設計のノウハウを学ぶことができます。

  • モード別のプロンプト戦略: 調査に特化した「Explore」、計画を練る「Plan」、実行を行う「Task」など、状況に応じて AI がどう振る舞いを変えているかの解明。
  • ツール呼び出しのガードレール: Bash コマンドの実行やファイル編集の際、安全性を保ちつつ正確に操作を行うための指示（命令文）の構成方法。
  • CLAUDE.md の参照ロジック: プロジェクト固有のルールを AI にどう優先させて読み込ませているかの詳細。
  • バージョン間での進化の追跡: アップデートごとにプロンプトがどう改善・洗練されているかの履歴（CHANGELOG）の確認。

目立つポイント

Claude Code は単一の長いプロンプトではなく、40 以上のモジュール化された断片（Fragments）を組み替えて動作しています。このリポジトリは、それらを一箇所に集約しており、複雑な「エージェント型アーキテクチャ」を構築したい開発者にとって、プロンプト・エンジニアリングの究極の参考資料となっています。

セットアップや使い方の流れ

  1. リポジトリを閲覧し、prompts/ などのディレクトリ内にある Markdown ファイルを確認します。
  2. 自分が作成している AI エージェントのプロンプト設計において、特定の機能（エラー解決、コードレビュー等）の指示出しを参考にします。
  3. CHANGELOG.md を読み、Anthropic のエンジニアがどのような問題（ハルシネーションや無限ループ等）にどう対処したかという知見を取り入れます。

どんな人向けか

  • 自律型 AI エージェントを開発中で、プロバイダーがどのようなプロンプトを書いているか知りたいエンジニア。
  • Claude Code の挙動が時々変わる原因を、内部の指示レベルで理解・デバッグしたいユーザー。
  • プロンプト・エンジニアリングの極致を学び、自分のスキルを向上させたい人。

注意点

これらはあくまで「抽出された参照用データ」であり、ここにあるファイルを編集しても手元の Claude Code の動作が変わるわけではありません。また、Anthropic 公式のリポジトリではないことに留意してください。

まとめ

claude-code-system-prompts は、AI の魔法の裏側にある「論理的な言葉の積み重ね」を明らかにするリポジトリです。エージェントの振る舞いを司る設計思想を学ぶことで、私たちは AI をより深く理解し、より高度に使いこなすことができるようになるでしょう。
