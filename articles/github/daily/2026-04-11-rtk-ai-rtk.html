これは何か

rtk は、開発者が日常的に使用する CLI コマンドの出力を、AI (LLM) にとって効率的な形式にフィルタリング・圧縮するための高性能 CLI プロキシです。Rust で開発されており、ターミナルでの開発体験を損なうことなく、Claude Code や Cursor といった AI エージェントが読み取るトークン量を劇的に削減します。

何ができるか

AI にとって冗長な進捗バー、繰り返されるログ、不要なメタデータなどを自動的に排除し、重要な情報のみを抽出して LLM コンテキストに渡します。Git (status, diff, log) はもちろん、cargo, vitest, pytest といったテストツール、ESLint や Ruff などのリンター、さらには AWS CLI や Docker まで幅広くサポートしています。

目立つポイント

最大の特徴は「人間と AI 両方への最適化」です。人間が見るターミナルには視覚的に美しくインタラクティブな出力を表示し続けつつ、AI に渡される時だけデータを極限まで圧縮します。Rust 製のためオーバーヘッドは 10ms 未満と極めて小さく、単一バイナリで動作するゼロ依存設計も魅力です。

セットアップや使い方の流れ

インストールはワンラインのスクリプトで完了します。

curl -fsSL https://get.rtk.ai | sh

インストール後、rtk init --global を実行することで、例えば Claude Code のようなツールから実行されるコマンドを自動的に rtk 経由に置き換えることができます。手動で使う場合は、コマンドの頭に rtk を付けるだけ（例：rtk git diff）で動作します。

どんな人向けか

Claude Code, Cursor, GitHub Copilot などの AI コーディングツールを頻繁に使用し、API のトークンコストを抑えたい開発者に最適です。また、大規模なプロジェクトで diff やログがコンテキスト制限に抵触しがちなチームにとっても、情報密度を保ちつつ制限を回避する強力な武器になります。

注意点

出力を最適化するために情報を削ぎ落とす性質上、非常に稀ではありますが、AI が解決に必要とする極めて詳細なメタデータまで削られてしまう可能性はゼロではありません。デバッグで完全な生ログが必要な場合は、フィルタリングされていない出力を確認するか、一時的にプロキシを介さず実行する必要があります。

まとめ

rtk は、AI との協調開発における「トークン効率」という現代的な課題を極めてエレガントに解決するツールです。トークン消費を最大 90% 削減できる可能性を秘めており、AI ネイティブな開発環境における必須のミドルウェアとなるポテンシャルを持っています。
