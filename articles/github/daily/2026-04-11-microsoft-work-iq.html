これは何か

microsoft/work-iq は、Microsoft 365 (Outlook, Teams, OneDrive, SharePoint) の仕事データと AI エージェントを橋渡しするための公式ツールです。MCP (Model Context Protocol) サーバーとして動作し、Claude Code や Cursor といった AI ツールが、あなたの「仕事の文脈」を直接参照できるように設計されています。

何ができるか

AI を通じて、過去のメールの要約、カレンダーの予定確認、最近編集したドキュメントの検索、さらには Teams でのやり取りに基づいた議論の整理などが可能になります。これにより、AI はただコードを書くだけでなく、「先週の会議での決定事項に基づいて実装を修正して」といった高度な文脈依存の指示を理解できるようになります。

目立つポイント

Microsoft 公式が提供する安心感と、オープン標準である MCP の採用が大きな特徴です。Graph API を介したセキュアなデータアクセスを維持しつつ、特定の AI プラットフォームに縛られない汎用性を備えています。また、GitHub Copilot CLI の拡張としても動作するため、ターミナル環境との親和性も抜群です。

セットアップや使い方の流れ

Node.js 環境があれば、npm を通じて簡単にインストールできます。

npm install -g @microsoft/workiq

MCP サーバーとして利用する場合は、Claude Desktop や VS Code の MCP 設定ファイルに npx -y @microsoft/workiq mcp を追記するだけです。初回実行時にブラウザで Microsoft 365 アカウントの認証を行うことで、AI があなたの業務データにアクセスできるようになります。

どんな人向けか

Microsoft 365 を業務基盤として使用している開発者やナレッジワーカーにとって、生産性を一段階引き上げる強力な武器になります。特に、プロジェクトの背景（メールや資料）と実装（コード）を一つの AI ワークフローで繋げたいエンジニアに強く推奨されます。

注意点

現在はパブリックプレビュー段階であり、仕様変更の可能性があります。また、組織環境での利用には、Microsoft 365 テナント管理者による API アクセスの承認が必要になる場合があるため、導入前に社内のポリシーを確認することをお勧めします。

まとめ

work-iq は、AI が「仕事」そのものを理解するためのミッシングピースを埋めるツールです。Microsoft 365 という広大な知識ベースを AI エージェントに開放することで、AI との協働は単なるタスク代行から、真の意味での「インテリジェントな相棒」へと進化します。
