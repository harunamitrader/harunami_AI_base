これは何か

Decepticon は、企業の防御体制を実戦形式でテストする「レッドチーム」の活動を、AI エージェントによって自律化した高度なセキュリティ・プラットフォームです。

単に CVE（脆弱性情報）をスキャンしてレポートを出すだけのツールとは異なり、偵察、脆弱性の特定、攻撃コードの実行、C2（指令・制御）サーバーの展開、ドメイン情報の収集、そして内部ネットワークへの横展開（ラテラルムーブメント）といった、攻撃者が辿る「キルチェーン」全体を自律的に遂行します。

何ができるか

  • 自律型攻撃シナリオの実行: 作戦計画（OPPLAN）に基づき、偵察から特権昇格までを人の介入なしに連続実行します。
  • 対話型ツールの操作: msfconsole や sliver-client といった対話が必要なセキュリティツールを、tmux 経由で人間のように操作します。
  • インフラの分離: 攻撃用の Kali Linux サンドボックス、C2 サーバー、監視システムが物理的・論理的に分離された安全なアーキテクチャで動作します。
  • プロフェッショナルな枠組み: 交戦規定（RoE）やデコンフリクション（衝突回避）計画など、実務的なレッドチーム運用のワークフローを強制します。

目立つポイント

最も特筆すべきは、AI が「ツールのプロンプト（msf6 > 等）」を検出し、次のコマンドを投げるという、徹底したインタラクティブ性の実現です。従来の AI エージェントが苦手としていた、一度起動して待ち受けるタイプの攻撃ツールを完全に使いこなします。

また、MITRE ATT&CK フレームワークと密に連携しており、各アクションがどのような攻撃手法に相当するかが明確にマッピングされます。これにより、防御側（ブルーチーム）はどの攻撃チェーンで検知が漏れたのかを正確に振り返ることが可能です。

セットアップや使い方の流れ

Docker と Docker Compose v2 があれば、ワンライナーでインストール可能です。 

curl -fsSL https://decepticon.red/install | bash
decepticon config  # Anthropic または OpenAI の API キーを設定
decepticon         # 起動

decepticon demo コマンドを実行すると、ターゲットとなる脆弱なサーバー（Metasploitable 2）を自動で立ち上げ、実際の攻撃フローをデモンストレーションしてくれます。

どんな人向けか

  • 継続的なセキュリティ評価を自律化したい企業のセキュリティ担当者（SOC/CSIRT）。
  • AI エージェントを実務的なレッドチーム運用に統合したいセキュリティエンジニア。
  • 高度なサイバー攻撃の挙動を安全な環境で学習・シミュレーションしたいプロフェッショナル。

注意点

Decepticon は極めて強力な「武器」にもなり得るツールです。必ず明示的な許可を得た所有システムやネットワーク上でのみ使用してください。無許可での使用は違法であり、開発者は一切の責任を負わないことが明記されています。

まとめ

Decepticon は「AI にハッキングができるか」という初期の問いを超え、「AI をどのようにプロフェッショナルなレッドチーム活動に組み込むか」という実運用フェーズへの回答となるプロジェクトです。単なる攻撃ツールではなく、防御側の能力を高めるための強固な「仮想ウイルス（Offensive Vaccine）」としてのビジョンを持っており、セキュリティと AI の融合における重要な試金石となるでしょう。
