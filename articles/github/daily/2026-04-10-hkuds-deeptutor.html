プロジェクトの概要

DeepTutor は、AI チャット、深掘り問題解決、クイズ生成、リサーチ、教材生成を一つの会話文脈で回せる学習支援プラットフォームです。 2026 年 4 月の v1 系ではアーキテクチャを全面刷新し、TutorBot、Co-Writer、Guided Learning、持続メモリなどを agent-native な設計に再編しています。

機能・できること

  • Unified Chat Workspace: Chat、Deep Solve、Quiz Generation、Deep Research、Math Animator を同じスレッドで切り替えられます。
  • Personal TutorBots: 個別のメモリと性格、スキルを持つ自律的な学習ボットを用意できます。
  • AI Co-Writer: Markdown 編集に AI 協調を組み込み、書き換えや要約を知識ベースと接続して行えます。
  • Knowledge Hub: PDF や Markdown を取り込み、RAG ベースの学習知識庫として再利用できます。

リポジトリの構成

フロント、バックエンド、CLI が分かれており、プロダクトとしての形がかなり明瞭です。

  • deeptutor/: サーバー側のコア実装。
  • deeptutor_cli/: CLI エントリポイント。
  • web/: Next.js ベースのフロントエンド。
  • scripts/、requirements/、tests/: 補助スクリプト、依存定義、テスト。

セットアップ・導入のイメージ

Docker かローカル構築の両方が用意されており、README では Python 3.11 以上を前提にしています。 `.env` では OpenAI、Anthropic、Gemini、Copilot、Ollama など多様な LLM・Embedding プロバイダを切り替えられます。

  1. リポジトリを clone し、pip install -e ".[server]" とフロント側の npm install を実行します。
  2. .env.example をコピーして LLM、Embedding、Web Search の接続先を設定します。
  3. ローカル起動後、チャット、知識ベース、TutorBot、Co-Writer などを用途別に使い分けます。

ターゲット層 / 注目ポイント

個人学習者だけでなく、教材作成、社内教育、RAG 付きナレッジ学習を同じ UI で回したいチームにも向いています。 とくに学習体験を「一問一答チャット」から「継続する個別エージェント」へ引き上げようとしている点が目立ちます。

制約・リスク

機能面は広い一方で、LLM、埋め込み、検索、フロント、バックエンドを全部組むため設定項目は多めです。 教育用途では回答の正確性検証、学習履歴の扱い、モデルごとのコスト管理も実運用の論点になります。

まとめ

DeepTutor は、教育アプリに AI を足したのではなく、学習そのものをエージェント中心に組み直そうとしているプロジェクトです。 OSS の学習支援基盤として見ても機能密度が高く、教育 × RAG × agent の交点を追うなら押さえておきたい一件です。
