これは何か

Claude Code や Cursor などの AI コーディングエディタが、タスクごとにコードベース全体を読み直してトークンを浪費する問題を解決するためのツールです。Tree-sitter を利用してコードの抽象構文木（AST）を解析し、関数、クラス、インポートの関係性をグラフ化して保持します。これにより、AI は「修正すべき箇所と、そこから影響を受ける最小限の範囲（Blast Radius）」だけをスマートに読み取れるようになります。

何ができるか

MCP（Model Context Protocol）サーバーとして動作し、AI エージェントに対して以下のような高度なコンテキスト提供を自動化します。

  • 影響範囲の特定： 特定のファイルを変更した際、どの関数やテストが影響を受けるかをグラフ上でトレース。
  • トークンの劇的削減： レビュー時で平均 6.8 倍、日常的なタスクで最大 49 倍のトークン消費を抑制。
  • セマンティック検索： コードの構造に基づいた高精度なノード検索。
  • 可視化： コードの依存関係を SVG や Obsidian vault 形式などで書き出し。

目立つポイント

特に巨大なモノレポにおいて威力を発揮します。何万ものファイルがあるプロジェクトでも、グラフ解析によって実際に AI が読むべきファイルを 10 数件程度に絞り込むことができ、コンテキストウィンドウの枯渇を防ぎます。また、23 以上の言語に加え、Jupyter ノートブック（.ipynb）にも対応している点が非常に実用的です。

セットアップや使い方の流れ

Python 3.10 以上がインストールされた環境で、以下の手順で導入できます。

  1. pip install code-review-graph（または pipx, uvx）
  2. code-review-graph install を実行して、使用中のエディタ（Claude Code/Cursor等）に MCP 設定を自動注入。
  3. プロジェクトディレクトリで code-review-graph build を実行し、ナレッジグラフを構築。
  4. エディタを再起動し、AI に「このプロジェクトのグラフを構築して」と指示するだけで、自動的に最適化された読み込みが開始されます。

どんな人向けか

  • AI エージェントのトークン使用量（およびコスト）を削減したい開発者。
  • 大規模なプロジェクトに従事しており、AI がコンテキストを正しく把握できずに困っている方。
  • コードの依存関係を視覚的に把握し、変更の影響を正確に見極めたいエンジニア。

注意点

初回ビルドには数秒（500ファイル程度で約10秒）かかります。その後はインクリメンタル（差分）更新に対応していますが、大幅なディレクトリ構造の変更などがあった場合は再ビルドが推奨されます。また、現時点では JavaScript や Go のコントロールフロー検出など、一部の言語固有の高度な機能については継続的な改善が進んでいる段階です。

まとめ

code-review-graph は、AI コーディングにおける「全読み」の非効率を構造化データで解決する、まさに AI 時代の「コードインデックス」と言える存在です。コスト削減だけでなく、AI が「本当に重要なコード」に集中できるため、回答の精度向上にも直結する強力なツールです。
