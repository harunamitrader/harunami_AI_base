これは何か

claude-token-efficient は、プロジェクト直下に置く 1 枚の CLAUDE.md で Claude Code の応答を短く保ちやすくする設定リポジトリです。README では、無駄な前置き、言い換え、不要な提案、過剰設計を抑えて、出力中心のワークフローでトークン消費を減らすことを狙っています。

何ができるか

導入は単純で、配布されている CLAUDE.md をプロジェクトルートに置くだけです。これにより、まず読んでから書く、簡潔に返す、大きなファイルをむやみに書き換えない、完了前に検証する、といった振る舞いを Claude Code に強く促せます。

目立つポイント

特徴は「短くしろ」だけでなく、どの冗長さを削るかがかなり具体的な点です。README の比較では説明系タスクで 50% から 75% 程度の語数削減例を示し、外部ベンチマーク欄では複数のコーディング課題で総コスト 17.4% 減という結果も紹介しています。加えて root の CLAUDE.md 自体は短く保ち、毎ターンの入力オーバーヘッドを抑える設計になっています。

セットアップや使い方の流れ

基本は README のとおり、対象プロジェクトのルートに CLAUDE.md を置いて Claude Code に読ませます。その後は普段どおり使えばよく、出力が長すぎるレビュー、説明、エージェント反復処理などで効果を見やすい構成です。必要に応じて利用者側の明示指示で詳細説明モードへ戻せる前提も置かれています。

どんな人向けか

多数の反復呼び出しを回す自動化、コード生成、レビュー運用で「答えの質より説明の長さ」にコストが乗っているチーム向けです。特に同じ形式の応答を何度も返させるワークフローでは、積み上がる余計な文言を減らす効果が出やすそうです。

注意点

README でも明記されている通り、この手法は毎ターン CLAUDE.md を入力コンテキストに載せるため、単発の短い質問では逆に損になりえます。また、構造化出力の厳密保証や大きな設計逸脱の防止は prompt だけで完全には担えないので、必要なら schema やフックで補強する前提で見た方が安全です。

まとめ

claude-token-efficient は、Claude Code を別物に作り替えるというより、冗長さの癖を小さいルールで矯正して運用コストを下げるための実務向け設定です。長い説明が価値にならない場面では、かなり導入しやすい改善案です。
