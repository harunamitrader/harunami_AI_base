これは何か

claude-subconscious は、Claude Code の下で Letta Agent を常駐させ、セッション transcript とコードベースを読みながら補助メッセージを返すデモアプリです。

README 冒頭でも本番利用向けではなく、実運用するなら Letta Code を使うよう明記されています。つまり製品というより、構成例として読む repo です。

何ができるか

Claude Code のセッション開始、プロンプト送信前、ツール実行前、応答停止時に hook を差し込み、背景側の Letta Agent が transcript を処理します。

その Agent は Read、Grep、Glob、Web 検索などを使い、コードや会話を見ながら記憶を更新し、次のプロンプト前に whisper 形式で助言を差し込めます。

目立つポイント

記憶を CLAUDE.md に書き戻さず、stdout 経由で注入する設計なので、既存の運用ファイルを汚さないのが面白いです。

複数セッションを 1 つの Letta Agent に束ねて記憶共有する説明や、SDK tools の read-only / full / off 切り替えまであり、背景エージェント設計の教材として密度があります。

セットアップや使い方の流れ

導入は plugin marketplace 経由か source install で行い、最低限 LETTA_API_KEY を設定します。必要に応じて LETTA_MODEL、LETTA_AGENT_ID、LETTA_SDK_TOOLS を追加します。

動作の中心は Claude Code hook と Letta Code SDK の組み合わせで、Stop hook 側では transcript を非同期 worker へ渡し、Claude 側を待たせない作りです。

どんな人向けか / 注意点

Claude Code に長期記憶や背景エージェントを足したい人、Letta を使った coding assistant の実験をしたい人、hook 設計を学びたい人に向いています。

ただし transcript とコード読み取りを伴うので、権限設計とプライバシー管理は必須です。README 自体も production 用ではないと明示しています。

まとめ

claude-subconscious は、Claude Code の外側にもう 1 層エージェントを置くと何ができるかを具体化した実験 repo です。2026-04-04 の Trending Rank 5 として見ても、背景メモリの設計例としてかなり示唆があります。
