プロジェクトの概要

rowboat は、メールや会議メモに接続し、長く使える知識グラフを構築するローカルファーストの AI coworker です。 生成 AI に一時的な会話をさせるのではなく、人、プロジェクト、意思決定、未解決事項を Markdown ベースの作業記憶として残す点が特徴です。

機能・できること

  • 知識グラフ構築: メール、会議、メモから人物、会社、トピック、決定事項を整理します。
  • 文脈付きアウトプット: 会議準備、メール下書き、資料、PDF スライドなどを既存文脈から生成できます。
  • Markdown vault: Obsidian 互換の plain Markdown と backlinks として記憶を確認・編集できます。
  • 拡張連携: Gmail、Calendar、Drive、音声入出力、Exa 検索、MCP / Composio 連携を追加できます。

リポジトリの構成

apps/ を中心にデスクトップ/アプリ層を持ち、docker-compose.yml や Dockerfile.qdrant からベクトル検索基盤も含む構成です。 google-setup.md には Google サービス接続の案内があります。

セットアップ・導入のイメージ

README では Mac、Windows、Linux 向けのダウンロードが案内されています。Google サービスに接続する場合は Gmail、Calendar、Drive の認可設定を追加し、任意で Deepgram、ElevenLabs、Exa、MCP サーバーを設定します。

  1. 公式サイトまたは GitHub Releases からアプリを入手します。
  2. Google setup に沿ってメールやカレンダーへの接続を設定します。
  3. 必要に応じて音声入力、音声出力、外部検索、MCP ツールを追加します。

ターゲット層 / 注目ポイント

会議、メール、ドキュメント作成が多い個人や小規模チームに向いています。 「会話履歴」ではなく「編集可能な知識グラフ」を中心に据えるため、AI 秘書を長期記憶付きの作業環境として運用したい人に刺さります。

制約・リスク

メールや会議メモを扱うため、認可範囲とローカルデータ管理が重要です。便利さを優先して接続先を増やしすぎると、秘密情報や個人情報の境界が曖昧になりやすい点には注意が必要です。

まとめ

rowboat は、AI coworker を単発のチャットではなく、日々の仕事の記憶層として捉え直すプロジェクトです。 Obsidian 互換の Markdown を土台にしているため、ブラックボックス化しにくい点も実用上の強みです。
