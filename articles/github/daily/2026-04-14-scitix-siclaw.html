これは何か

障害調査とインフラ診断をプログラム制御する AI SRE エージェント・プラットフォームです。 既存のインフラに対して「読み取り専用（Read-only）」で動作し、エビデンスの収集、仮説の立案、その後の検証までを自動で行います。 「インフラを壊すのが怖い」「でも AI に調査を手伝ってほしい」という SRE チームの要求に応えるツールです。

主な特徴と機能

  • 深い調査ワークフロー: エビデンスの収集、仮説の構築、検証、そして最終的な根本原因分析。AI がこれらを自律的に繰り返すことで、人間より圧倒的に早く障害箇所を突き止めます。
  • 安全な診断プロセス: デフォルトで読み取り専用の操作に限定。デプロイや設定変更を行わずに、ログやメトリクスを縦横無尽に調査します。
  • MCP (Model Context Protocol) 対応: 外部の監視ツールやデータベースと MCP 経由で接続し、AI が自ら情報を取ってこれる拡張性を備えています。
  • マルチチャネルの利便性: 手軽な TUI（ターミナル）、チームで共有できる Web UI、さらに Slack や Discord などのチャットツールからも指示・報告が可能です。

なぜ重要か

大規模な Kubernetes クラスターなどの障害調査では、何が起きているかを確認するだけで膨大な時間がかかります。 Siclaw は、「CrashLoopBackOff の原因を調べて」といった自然言語の指示を受けると、AI が自ら kubectl やログを叩き、過去のナレッジを活かして原因を特定します。 調査の手順（スキル）は再利用可能なランブックとして登録でき、チーム全体でノウハウを共有できる点も大きなメリットです。

技術スタック

TypeScript で記述されており、Node.js 22 以上で動作します。 Kubernetes へのデプロイも Helm チャートで提供されており、エンタープライズ環境での利用も考慮されています。

まとめ

scitix/siclaw は、DevOps/SRE の定型的な調査作業を AI に任せるための基盤です。 「調査は AI、判断は人間」という安全な協力体制を築くための、現在もっとも注目すべき SRE エージェントの一つと言えるでしょう。
