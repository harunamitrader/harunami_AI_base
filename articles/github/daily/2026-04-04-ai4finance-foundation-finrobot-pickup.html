これは何か

finrobot は、金融向けに特化した AI agent platform です。README では FinGPT の単一モデル的な使い方を超えて、LLM、reinforcement learning、quantitative analytics を組み合わせ、investment research automation、algorithmic trading、risk assessment を扱う土台として説明されています。

何ができるか

特に分かりやすいのは finrobot_equity 系のフローで、財務データ取得、3 年予測、DCF valuation、peer comparison、investment thesis、risk assessment、そして multi-page の HTML/PDF equity research report 生成までを一連で回せます。単なるチャット agent ではなく、証券リサーチ作成の pipeline をそのままコード化した印象です。

目立つポイント

README では全体を 4 層で整理していて、Financial AI Agents、Financial LLM Algorithms、LLMOps/DataOps、Foundation Models という構造がはっきりしています。さらに Smart Scheduler によって task ごとに適切な agent や model を割り当てる思想が示されており、finance 特化の multi-agent orchestration として筋が通っています。

セットアップや使い方の流れ

基本は Python 3.10 環境を作り、repo を clone して pip install -e . か pip install -U finrobot で導入します。その後 OAI_CONFIG_LIST と config_api_keys を用意し、tutorial notebook を触るか、finrobot_equity の web UI を deploy.sh または python run_web_app.py で起動する流れです。

どんな人向けか

financial data ingestion からレポート生成までを agent workflow で統合したい人に向いています。quant、fintech、投資リサーチ自動化、あるいは社内向け equity research assistant を組みたいチームには、かなり参考になる構成です。

注意点

実運用には OpenAI に加えて Finnhub、Financial Modeling Prep、SEC API など複数の API key が必要です。また、出力される分析やレポートは投資判断支援の自動化土台として見るべきで、そのまま売買判断を委ねる前提のものではありません。

まとめ

finrobot は、finance 領域で agent orchestration を組みたい人向けの OSS としてかなり完成度が高いです。特に「複数 agent で財務分析を回し、そのままリサーチレポートに落とす」という出口が明確なので、金融向け AI workflow の具体例を探しているなら見る価値があります。
