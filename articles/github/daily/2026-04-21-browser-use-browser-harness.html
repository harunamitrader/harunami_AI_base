これは何か

browser-harness は、LLM エージェントがブラウザと対話するための、極めて軽量かつ野心的な接続プラットフォームです。人気プロジェクト「browser-use」のチームによって開発されており、エージェントが Web ページを操作するだけでなく、必要に応じて自分自身の「ヘルパー関数」を書き換えて機能を拡張する「自己修復（Self-healing）」機能を核としています。

何ができるか

Chrome DevTools Protocol (CDP) に直接接続することで、エージェントはブラウザのあらゆる機能を制限なく制御できます。単なるスクレイピングや自動化を超え、エージェント自身がタスク実行中に「特定の操作を簡単に行うためのツールがない」と判断した場合、ソースコード（helpers.py 等）を自律的に編集し、新しい能力を手に入れることができます。これにより、プログラマーが事前に定義した枠組みを超えたタスク遂行が可能になります。

目立つポイント

最大の特徴は、その「薄さ」と「進化の自由度」にあります。複雑な抽象化層を排除し、LLM がブラウザの状態（DOM、アクセシビリティツリー等）と直接向き合えるように設計されています。また、Browser Use Cloud との連携により、ローカル環境を汚さずにリモートブラウザを即座に立ち上げ、並列実行できる利便性も備えています。エージェントが「道具を使って作業する」のではなく、「作業しながら道具を作る」というパラダイムシフトを体現しています。

セットアップや使い方の流れ

Python 環境（3.11以降推奨）で pip install browser-harness を実行することで利用可能です。ブラウザが必要な場合は playwright install chromium でインストールし、最小限のスクリプトでエージェントをブラウザに接続できます。Browser Use Cloud を利用する場合は、API キーを設定するだけで、設定済みのブラウザデーモンをリモートから操作できるようになります。

どんな人向けか

Claude Code や Codex などの自律型コーディンエージェントを活用している開発者、また独自のブラウザ操作エージェントを構築したいリサーチエンジニアに最適です。固定のルールエンジンの限界に突き当たっている方や、エージェントに究極の自律性を持たせたいと考えている方にとって、これ以上ない強力な足場となるでしょう。

注意点

エージェントが自身のソースコードを書き換えるという特性上、予期しない動作やバグの混入に対する保護措置が重要になります。実行環境は必ず隔離（サンドボックス化）し、エージェントの行動を監視・制限するポリシー（Policy）を適切に定義することが推奨されます。また、現時点では進化のスピードが非常に早いため、頻繁なアップデートが予想されます。

まとめ

browser-harness は、AI エージェントが「不完全な環境」に適応し、自律的に成長するための画期的なアプローチを提供します。Web オブザーバビリティと自己進化を組み合わせたこの基盤は、エージェントが単なる「実行者」から「創造者」へと進化する第一歩となるかもしれません。ブラウザという広大な海を航海するエージェントたちのために、新しい羅針盤が登場しました。
