これは何か

AI 研究・開発の第一人者である Andrej Karpathy 氏が指摘した、「LLM がコーディング中に陥りがちな落とし穴」を回避するための具体的な行動指針をまとめたリポジトリです。Claude Code をはじめとする AI エージェントの挙動を制御するための CLAUDE.md のベースを提供しています。

目玉となる特徴

最大の特徴は、AI が勝手に行う「推測」や「過剰なコード生成」を厳格に制限する 4 つの原則です。これにより、AI はより確実に、かつ余計なことをしない「ベテランエンジニア」のように振る舞うようになります。

もっと詳しく知る

このプロジェクトが提唱する **4つの原則の詳細解説** や、そのまま使える **日本語版 CLAUDE.md テンプレート** など、踏み込んだ内容は以下の詳細ページにまとめています。

👉 Karpathy 流 CLAUDE.md 実践ガイド・詳細解説を見る

どんな人向けか

Claude Code などの AI コーディングツールを利用しており、生成されるコードの「肥大化」や「勝手な変更」に悩んでいる開発者に最適です。
