プロジェクトの概要

seomachine は、Claude Code を単なるチャット補助ではなく、長文 SEO コンテンツ制作の専用オペレーティング環境として構成したリポジトリです。 キーワード調査、競合分析、下書き、リライト、公開前最適化までを、命令セットとコンテキストファイルで再現可能なワークフローに落とし込んでいます。

機能・できること

  • 専用コマンド群: /research、/write、/rewrite、/optimize など、記事制作工程ごとの入口が用意されています。
  • 役割分担されたエージェント設計: コンテンツ分析、SEO 最適化、見出し生成、内部リンク設計、CRO 分析などの役割を分離しています。
  • マーケティング向けスキル同梱: コピー、A/B テスト、価格訴求、メール導線など 26 種の実務スキルを前提にしています。
  • 分析データ連携: Google Analytics 4、Search Console、DataForSEO を組み合わせて実測値ベースで記事改善できます。

リポジトリの構成

ルート直下にワークスペースの材料が整理されており、記事生成だけでなく運用プロセス全体を管理しやすい構成です。

  • .claude/、CLAUDE.md: Claude Code に読み込ませる指示と環境定義。
  • context/: ブランドボイス、社内ルール、内部リンク戦略、ターゲットキーワードなどの運用知識。
  • research/、drafts/、published/: 調査から公開までの成果物置き場。
  • research_*.py 群: SERP 分析、競合比較、優先順位付けなどの補助スクリプト。

セットアップ・導入のイメージ

前提は Claude Code と Anthropic API の利用です。さらに Python 依存を入れることで、外部データを使った分析まで回せます。

  1. リポジトリを clone し、pip install -r data_sources/requirements.txt で分析系依存を導入します。
  2. context/brand-voice.md や context/target-keywords.md などのテンプレートを自社向けに埋めます。
  3. Claude Code で開き、専用コマンドを使って調査、構成、執筆、最適化を順に流します。

ターゲット層 / 注目ポイント

コンテンツマーケティングを AI に外注するのではなく、自社の文脈を持った半自動制作ラインとして運用したいチームに向いています。 特に「AI ライター」ではなく「AI 編集部」を作る方向性が明確で、Claude Code ワークスペースの実務化サンプルとしても見やすい構成です。

制約・リスク

効果はコンテキストファイルの質に大きく依存します。ブランド情報や内部リンク設計が薄い状態では、汎用的な記事生成に戻りやすいです。 また、外部分析 API を前提にした運用では、料金や認証設定の管理も別途必要になります。

まとめ

seomachine は、SEO 記事制作を Claude Code の再利用可能な作業系として固めたい人にとって実践度の高いリポジトリです。 文章生成そのものより、調査と改善を含む編集フローをどう定義するかに価値があります。
