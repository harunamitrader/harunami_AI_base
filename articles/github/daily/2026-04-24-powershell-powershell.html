これは何か

Microsoftがオープンソースとして公開している、強力なタスク自動化フレームワークおよびコマンドラインシェルです。.NETをベースにしており、従来のテキストベースのシェルとは異なり「オブジェクト」を直接パイプラインで受け渡すことができるのが最大の特徴です。

何ができるか

単なるコマンド実行にとどまらず、複雑なスクリプト記述、システム管理、クラウド（Azure/AWS等）の操作が可能です。Windows PowerShellの後継として、現在はLinuxやmacOSでも全く同じコマンド体系で動作するため、クロスプラットフォームなインフラ構築に威力を発揮します。

目立つポイント

最近のトレンド入りは、AIエージェントによるターミナル操作の「標準インターフェース」としての地位を確立しつつあることが一因です。オブジェクト形式で結果を返せるため、LLMがパース（解析）しやすく、Claude CodeやGemini CLIなどの自律エージェントが「環境を理解し、操作する」ための強力な武器となっています。

セットアップや使い方の流れ

Windows 10/11には標準搭載されていますが、最新のオープンソース版（PowerShell 7.x）は公式リポジトリからダウンロードしてインストールします。Linuxであればパッケージマネージャー（apt/yum等）経由で、macOSであればHomebrewから即座に導入可能です。

どんな人向けか

インフラエンジニア、SRE、あるいは複数のOSが混在する環境で開発を行うプログラマーに最適です。また、最近ではプロンプトエンジニアリングの一環として、AIにシステム操作を教え込む際の「言語」として習得する層も増えています。

注意点

従来のWindows専用「Windows PowerShell (5.1)」と、クロスプラットフォームな「PowerShell (7.x)」では、一部のモジュールの互換性がない場合があります。新機能の恩恵を受けるには、可能な限りPowerShell 7系への移行を推奨します。

まとめ

PowerShellは、システム管理者の道具箱から「AI時代の自動化OS」へとその役割を広げています。テキストではなくオブジェクトで会話するその設計は、まさに構造化データを好む現代のAIエージェントにとって最適な進化を遂げたと言えるでしょう。
