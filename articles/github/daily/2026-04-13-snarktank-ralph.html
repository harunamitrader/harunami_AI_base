プロジェクトの概要

ralph は、PRD の user story がすべて passes: true になるまで、AI コーディングツールを繰り返し起動する自律ループです。 各 iteration は新しい AI インスタンスで実行され、記憶は git history、progress.txt、prd.json に残します。

機能・できること

  • PRD 駆動: Markdown の PRD を prd.json に変換し、user story 単位で進捗を管理します。
  • 反復実行: 最高優先度の未完了 story を選び、実装、品質チェック、commit、進捗更新を繰り返します。
  • 文脈リセット: 各 iteration を fresh context にすることで、巨大な作業を小さく分割して進めます。
  • Amp / Claude Code 対応: Amp CLI と Claude Code の両方で動き、Claude Code Marketplace plugin としても導入できます。

リポジトリの構成

中核は ralph.sh の bash loop です。prompt.md、CLAUDE.md、prd.json.example、skills/、.claude-plugin/、flowchart/ が含まれ、実行ループとスキル配布の両方を扱います。

セットアップ・導入のイメージ

プロジェクトへ ralph.sh とプロンプトテンプレートをコピーするか、Amp / Claude Code の skills として導入します。 Claude Code では marketplace を追加し、ralph-skills を install して PRD 作成と変換を使えるようにします。

  1. scripts/ralph/ に ralph.sh とプロンプトを配置します。
  2. /prd で PRD を作成し、/ralph で prd.json に変換します。
  3. ./scripts/ralph/ralph.sh --tool claude のように実行して反復させます。

ターゲット層 / 注目ポイント

AI コーディングを一発生成ではなく、PRD、品質チェック、git commit、学習ログを組み合わせた継続ループとして運用したい開発者に向いています。 AGENTS.md へ学びを残す設計により、次の iteration や将来の人間開発者にも知見が残る点が特徴です。

制約・リスク

PRD の粒度が大きすぎると、AI が 1 context で完了できずに品質が落ちます。typecheck、test、CI のようなフィードバックループを必ず用意し、失敗した変更が反復で増幅されないようにする必要があります。

まとめ

ralph は、AI コーディングを「新しい文脈で何度も試す」ための最小構成に近いプロジェクトです。 PRD と git を記憶層にする発想が明快で、長い作業を安全に分割したいチームにとって参考になります。
