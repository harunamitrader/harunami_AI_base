これは何か

Robusta KRR (Kubernetes Resource Recommender) は、Kubernetes クラスター内のリソース割り当てを最適化するための CLI ツールです。 既存の Prometheus から Pod の実際の使用状況データを収集し、CPU およびメモリの適切なリクエスト値（requests）と制限値（limits）を推奨します。

何ができるか

  • リソースの使用状況分析： Prometheus の履歴データに基づき、CPU とメモリの利用実態を詳細に解析します。
  • 最適化パラメーターの推奨： 無駄なリソース確保を減らし、逆に不足している箇所を補うための具体的な数値を提示します。
  • 多様な統合： Prometheus だけでなく、Thanos、Victoria Metrics、Managed Prometheus (Google/AWS/Azure) など、多くのデータソースに対応しています。
  • レポート出力： CLI での結果表示に加え、Slack 通知や Web UI での可視化も可能です。

目立つポイント

最大の特徴は、**「エージェントレスで即座にインフラの『無駄』を可視化できる点」**です。 クラスター内に常駐するエージェントをインストールする必要がなく、ローカル端末から CLI を実行するだけで即座に結果が得られます。 また、なぜその数値が推奨されたのかをグラフで確認できる説明可能性を備えており、インフラ担当者が自信を持ってリソース設定を変更できるよう支援します。

セットアップや使い方の流れ

Homebrew などを使用して brew install krr でインストールします。 クラスターに接続可能な環境で krr simple を実行すると、現在の割り当て量と推奨値、およびその差分が表示されます。 提示された数値をマニフェストファイルに反映させることで、最適化が完了します。

どんな人向けか

クラスターのクラウドコストが高騰しており、コスト削減の糸口を探している SRE やインフラエンジニアに最適です。 Resource Quota や HPA の設定値の根拠に悩んでいる開発チームにとっても、データに基づいた運用を実現するための強力な味方になります。

注意点

推奨の精度は Prometheus に蓄積された履歴の長さに依存するため、直近の短時間だけのデータではスパイク負荷を見落とす可能性があります。 最終的な判断は、アプリケーションの特性（起動時の負荷特性など）を考慮した上で人間が行う必要があります。

まとめ

krr は、Kubernetes のリソース管理という地味ながら難易度の高い課題に対して、既存のモニタリングデータを活用してシンプルに解を示してくれる優れたツールです。 まずは一度自社のクラスターでスキャンを実行し、どれほどのコスト削減の余地が潜んでいるかを確認してみる価値があります。
