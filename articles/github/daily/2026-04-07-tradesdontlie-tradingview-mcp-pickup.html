tradingview-mcp

TradingView Desktop を AI の「目と手」に変え、チャート分析と Pine Script 開発を効率化する MCP サーバー。

記事作成日 2026-04-07
種別 Pickup
対象 repo tradesdontlie/tradingview-mcp

これは何か

tradingview-mcp は、Claude Code などの AI エージェントから、ローカルで動作している TradingView Desktop を直接操作・分析するための MCP（Model Context Protocol）サーバーです。 開発者やトレーダー向けに、チャート分析から Pine Script の開発までを AI との対話で完結させる基盤を提供します。

何ができるか

  • チャート操作・閲覧: AI が銘柄切り替え、時間軸変更、特定の期間へのズーム、インディケーターの追加などを自律実行。
  • 視覚的分析: チャートのスクリーンショットを自動撮影し、AI がテクニカル的な形状や指標の動きを視覚的に解析。
  • Pine Script 開発支援: スクリプトの記述、コンパイル、チャートへのインジェクト、デバッグを AI がフルサポート。
  • アノテーション管理: トレンドライン、図形の描画、価格アラートの作成・削除をコマンド一つで制御可能。

目立つポイント

最大の強みは、Chrome DevTools Protocol (CDP) を介して **「デスクトップアプリ」** と直接リアルタイム通信を行う点です。 ブラウザ版だけでなく、機能の多いデスクトップ版を AI の「道具」として持たせることができ、リプレイ練習の操作や複雑な Pine Script の検証作業を大幅に自動化できます。

セットアップや使い方の流れ

  1. TradingView Desktop を起動し、リモートデバッグ（CDP）を有効にしておきます。
  2. npm install -g tradingview-mcp 等でインストール（リポジトリ構成に基づく）。
  3. Claude Desktop や Claude Code の config.json に MCP サーバーとして登録。
  4. AI に対し「BTCUSDの1時間足を表示して、スクショを撮って分析して」のように指示します。

どんな人向けか

  • 日々のチャート分析やトレーディングワークフローを、AI を使って効率化・自動化したいトレーダー。
  • Pine Script の開発効率を上げたい、あるいはデバッグを AI に手伝わせたい開発者。
  • Claude Code などの MCP 対応ツールを、金融分析の実用的なコパイロットとして活用したい人。

注意点

ローカルの TradingView Desktop アプリとの接続が必要なため、接続設定（ポート番号など）が正しく構成されている必要があります。また、操作の自動化による誤発注などを避けるため、実際の取引連携時には十分な確認が推奨されます。

まとめ

tradingview-mcp は、TradingView という強力なチャートツールを AI に開放するブリッジです。 これまで「人間が見て判断する」しかなかったチャート上の視覚情報や Pine Script の運用を、AI との共同作業に変える非常にポテンシャルの高いツールと言えます。
