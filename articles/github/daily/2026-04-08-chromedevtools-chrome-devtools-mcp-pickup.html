chrome-devtools-mcp

Google 公式の MCP サーバーが登場。AI エージェントに Chrome の「目」と「手」を与え、ブラウザを自在に操作・デバッグ可能に。

記事作成日 2026-04-08
種別 Pickup
対象 repo ChromeDevTools/chrome-devtools-mcp

これは何か

Google Chrome DevTools チームが開発した、Model Context Protocol (MCP) サーバーの公式実装です。Claude や Cursor といった AI エージェントが、実行中の Google Chrome インスタンスを直接制御し、Web ページの閲覧、デバッグ、パフォーマンス計測などを行うためのブリッジとなります。

何ができるか

このサーバーを導入することで、AI エージェントはブラウザを介して以下のタスクを自律的に実行できるようになります。

  • ブラウザ操作: 指定した URL への移動、テキスト入力、ボタンのクリック、スクロール、スクリーンショットの撮影。
  • DOM 解析とデバッグ: ページの構造（DOM）の読み取り、コンソールログの取得、ネットワーク通信の監視。
  • ストレージ操作: Cookie やローカルストレージの読み書き。
  • パフォーマンス計測: Google 公式ツールならではの強力なトレース機能を使用して、Web アプリのボトルネックを特定。

目立つポイント

これまで AI によるブラウザ操作の多くは Playwright などの外部ライブラリを介したスクリプト生成に頼ってきましたが、この MCP サーバーは Google 公式による Chrome DevTools Protocol (CDP) の直接利用を前提としています。これにより、エージェントはブラウザの「中身」をより正確かつリアルタイムに把握でき、より信頼性の高い Web オートメーションとデバッグが可能になります。

セットアップや使い方の流れ

基本的な導入ステップは以下の通りです。

  1. Chrome を --remote-debugging-port=9222 フラグを付けて起動し、リモートデバッグを許可します。
  2. AI ツールの MCP 設定（例: claude_desktop_config.json）に chrome-devtools-mcp を追加します。
  3. エージェントに対して「ブラウザを開いて example.com を調査して」といった指示を出します。
  4. エージェントが MCP ツール経由で Chrome を操作し、結果をあなたに報告します。

どんな人向けか

  • 複雑な Web アプリケーションのデバッグやテストを AI に手伝わせたい開発者。
  • Web サイトから動的に情報を収集（スクレイピング）する高度なエージェントを構築したい人。
  • Playwright などの設定なしで、使い慣れた Chrome を AI から直接操りたい人。

注意点

AI エージェントにブラウザの完全な操作権限を与えるため、セキュリティ上のリスクを認識し、信頼できる環境でのみ使用することが推奨されます。また、現時点では一部の機能が実験的な段階にある可能性があります。

まとめ

Google が MCP エコシステムに正式に参入し、ブラウザという最も重要な開発ターゲットを AI に開放したことは大きな前進です。chrome-devtools-mcp は、Web 開発における「AI とブラウザの完全な統合」を実現する強力な武器となるでしょう。
