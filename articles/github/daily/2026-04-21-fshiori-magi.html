これは何か

新世紀エヴァンゲリオンに登場するスーパーコンピュータ「MAGI」のコンセプトをLLMで再現した意思決定エンジンです。単一の高度なモデルに頼るのではなく、異なる役割（ペルソナ）を与えられた3つのLLMが、同じ問いに対して議論・投票・相互批判を行うことで、より信頼性の高い「集団的知性」を導き出します。

何ができるか

「科学者（メルキオール）」「母（バルタザール）」「女（カスパー）」といった独立した視点から回答を生成し、それらを戦わせることで単一モデルでは気づけない論理の穴やハルシネーションを補完します。高度な議論を必要とする戦略策定、コードレビュー、複雑なリサーチなどに最適化された複数の動作モード（投票、批判、適応型など）を備えています。

目立つポイント

最も注目すべき点は、格安な小規模モデルを組み合わせることで、Claude Sonnet クラスの最高峰モデルに匹敵する精度（MMLUベンチマークで実証済み）を叩き出せるコストパフォーマンスの高さです。また、NERVの指令室を模したリアルタイムGUIダッシュボードも搭載されており、AIエージェント同士の「思考の競合」を視覚的に楽しむこともできます。

セットアップや使い方の流れ

Python環境で pip install magi-system を実行して導入します。OpenRouterなどのAPIキーを設定後、CLIから magi ask "問い" を実行するだけで、3つのモデルによる評議が始まります。magi presets で用途に応じたペルソナセットを指定したり、magi dashboard でWeb UIを起動してデバッグや分析を行うことが可能です。

どんな人向けか

AIの回答の正確性を極限まで高めたい開発者やリサーチャー、またはエージェント同士による多角的な議論プロセスをワークフローに組み込みたい技術者向けです。特に、高価な最新モデルを使い続けるコストを抑えつつ、複数の安価なモデルで同等以上の成果を得たいプロジェクトに効果的です。

注意点

3つのモデルを同時に動かすため、APIトークンの消費量は単一モデル実行時の約3倍以上になります。また、議論（批判・修正）のラウンド数を増やすほど精度は向上しますが、最終的な結論が出るまでの待機時間は長くなる傾向があります。

まとめ

magiは、単なる「多数決」ではなく「構造化された不一致」を管理するユニークなエージェント基盤です。AIの自律的な議論を通じて知を磨き上げるアプローチは、今後の複雑なタスク自動化において有力な選択肢となるでしょう。何より、あのMAGIを自身のPCで稼働させる体験は、技術的にも感覚的にも刺激的です。
