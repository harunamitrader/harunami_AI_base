プロジェクトの概要

Archon は、AI コーディングエージェントの作業を deterministic に近づける workflow engine です。 「バグを直して」と頼むだけでは毎回手順がぶれやすい問題に対し、計画、実装、テスト、レビュー、PR 作成を YAML の DAG として定義します。

機能・できること

  • YAML ワークフロー: planning、implementation、validation、review、PR creation を明示的なノードとして書けます。
  • worktree 分離: 実行ごとに専用 git worktree を作り、複数タスクの並列実行を衝突しにくくします。
  • AI と決定的処理の分離: bash、テスト、git 操作などは deterministic node、判断や生成は AI node として分けられます。
  • 複数 UI: CLI、Web UI、Slack、Telegram、GitHub などから同じ workflow を呼び出せます。

リポジトリの構成

.archon/ にデフォルト workflow、packages/ に主要実装、auth-service/ や deploy/ に認証・展開関連が含まれます。 Web UI、CLI、platform adapter、orchestrator、workflow executor、SQLite / PostgreSQL という構成です。

セットアップ・導入のイメージ

README では Bun、Claude Code、GitHub CLI を前提にしたフルセットアップと、CLI だけを入れるクイックインストールが案内されています。 導入後は対象プロジェクトで Claude Code を起動し、「Use archon to fix issue #42」のように workflow を呼び出します。

  1. git clone https://github.com/coleam00/Archon で取得します。
  2. bun install 後、Claude Code から setup wizard を実行します。
  3. 対象リポジトリに Archon skill を入れ、workflow を実行します。

ターゲット層 / 注目ポイント

AI コーディングを個人の手癖ではなく、チームの標準手順として運用したい開発者に向いています。 既定 workflow が多く、issue 修正、idea-to-PR、PR review、conflict 解消、refactor などのよくある流れをすぐ試せます。

制約・リスク

workflow を固定化できる一方で、悪い手順を YAML 化すると失敗も再現可能になります。まずは小さなタスクで validation gate と権限境界を設計し、チームのレビュー基準に合わせて調整する必要があります。

まとめ

Archon は、AI コーディングの「毎回違う」を減らすための実務寄りツールです。 エージェントの賢さだけに頼らず、開発プロセスそのものをコード化する点が評価されています。
