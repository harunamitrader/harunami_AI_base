Cua

AIが「目」と「手」を持ってPCを操作するbox。デスクトップ全体を制御するComputer-Use Agent（CUA）のための、安全な実行・開発インフラ。

これは何か

AIエージェントがmacOS、Windows、LinuxなどのデスクトップOSを直接操作できるようにするためのオープンソース・インフラストラクチャです。「Computer-Use Agent (CUA)」という概念に基づき、AIが画面の視覚情報を読み取り、マウスを動かし、キーボードを入力するためのサンドボックス環境、SDK、およびベンチマークを提供します。

何ができるか

開発者はCuaを使用することで、AIに「ブラウザを開いて航空券を予約して」「Excelからデータを抽出してSlackに投稿して」といった、GUIを伴う複雑なタスクを自律的に実行させることができます。特に「Lume」という仮想化レイヤーを採用しており、ホストOSを汚さず、安全に隔離された環境（VM）の中でAIにデスクトップを操作させることが可能です。

目立つポイント

最大の特徴は、AIが「人間のカーソルを奪わずに」バックグラウンドでネイティブアプリを操作できる点です。また、macOSのネイティブアプリを駆動する専用ドライバや、AIが視覚情報を正確に認識するための座標変換・アクション抽出ロジックが高度に統合されています。さらに、MCP（Model Context Protocol）に対応しており、既存のAIクライアントから即座に「デスクトップ操作能力」を呼び出せます。

セットアップや使い方の流れ

基本的にはCLIツールとして、またはPython/TypeScript SDKとして導入します。`cua run` コマンドを使用すると、隔離されたデスクトップ環境が立ち上がり、AIエージェントはその中でスクリーンショットを撮りながらタスクを完遂します。LiteLLMを内蔵しているため、OpenAI、Anthropic、Googleなど、好みのLLMをバックエンドに使用できます。

どんな人向けか

ブラウザの自動化（Playwright等）を超えて、デスクトップアプリ（Slack、Discord、Zoom、デザインツール等）の操作をAIに任せたいと考えているエンジニアや、自律型エージェントのプロダクトを開発しているスタートアップに最適です。

注意点

AIにデスクトップ操作を許可することはセキュリティ上のリスクを伴います。必ずCuaが提供するサンドボックス（VM/Docker）環境で実行することが強く推奨されます。また、画面認識の精度は使用するモデル（Claude 3.5 Sonnet 等のビジョン能力が高いもの）に依存します。

まとめ

Cuaは、AIエージェントに「コンピュータを人間のように扱う能力」を与える強力な基盤です。サンドボックス化された安全な実行環境と、洗練されたSDKにより、SFのような「自律的なPC操作エージェント」の実装が、今や現実のエンジニアリングの範疇となっています。
