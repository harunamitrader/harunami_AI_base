これは何か

Everywhere は、デスクトップ環境に深く統合された AI アシスタントです。最大の特徴は、ユーザーが現在見ている画面の内容を「コンテキスト」として即座に理解できる点にあります。スクショを撮ったりテキストをコピーしたりする手間なく、ショートカットキー一つで AI を呼び出し、現在の作業内容に基づいた具体的な支援を受けることができます。

何ができるか

  • コンテキスト認識対話： 画面上のエラーメッセージ、Web 記事、編集中のコードやメールなどを AI が自動で読み取って回答。
  • マルチ LLM 対応： OpenAI、Gemini、Claude、DeepSeek、さらにローカルの Ollama など、主要なモデルを切り替えて利用可能。
  • MCP ツールの統合： Model Context Protocol (MCP) を通じて、外部ツールやデータとシームレスに連携。
  • クロスプラットフォーム： Windows および macOS に対応（Linux は近日対応予定）。
  • UI オートメーション： 画面の構造を理解し、対話を通じて操作や要約を効率化。

目立つポイント

既存の AI チャットツールが「ブラウザや専用アプリの中」で完結していたのに対し、Everywhere は「OS のレイヤー」で動作します。Avalonia を使用した洗練された UI は、デスクトップのどこにでも違和感なく馴染み、必要なときにだけ現れる「透明なアシスタント」のような体験を提供します。また、企業向けの Business Source License 1.1 を採用しており、開発の持続可能性も考慮されています。

セットアップや使い方の流れ

  1. GitHub の Releases ページから、OS に合ったインストーラー（Windows は .exe、macOS は .pkg）をダウンロードしてインストールします。
  2. アプリを起動し、使用したい AI モデルの API キー（または Ollama のローカル接続設定）を入力します。
  3. ショートカットキー（デフォルト設定）を押すと、画面上に Everywhere の入力窓が現れます。
  4. 「このエラーの原因は？」「このメールを丁寧に書き直して」など、画面を見せながら指示を送るだけで AI が実行します。

どんな人向けか

  • 作業中のコンテキストスイッチ（アプリの切り替え）を最小限に抑えたい開発者やクリエイター。
  • ブラウザ、エディタ、ターミナルなど、複数のアプリを跨いで AI の支援を受けたい人。
  • 特定の MCP ツールを活用して、自分専用の業務自動化環境を構築したいパワーユーザー。

注意点

画面上の情報を読み取る性質上、機密情報が表示されている環境での利用には配慮が必要です（設定でプライバシーポリシーを確認することをお勧めします）。また、記事執筆時点（2026年3月〜4月）で Linux への対応は開発中となっています。

まとめ

Everywhere は、AI が「ツールの外側」に飛び出し、デスクトップ全体をキャンバスにするための強力な一歩です。単なるチャットボットではなく、視覚情報を共有する「同僚」のような存在を目指しており、MCP への対応によってその拡張性は無限に広がっています。
