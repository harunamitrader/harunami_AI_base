プロジェクトの概要

VoxCPM は、離散トークンを介さずに連続的な音声表現を直接生成する tokenizer-free TTS プロジェクトです。 最新の VoxCPM2 では 30 言語対応、48kHz 出力、自然言語からの声質設計、参照音声を使った制御可能なボイスクローニングまで含め、研究寄りと実用寄りの中間を強く狙っています。

機能・できること

  • 多言語音声合成: 日本語を含む 30 言語と複数の中国方言に対応しています。
  • Voice Design: 参照音声なしで「若い女性、やさしい声」などの自然言語記述から話者設計が可能です。
  • Controllable Voice Cloning: 参照音声の声質を保持しつつ、速度や感情、スタイルを追記で調整できます。
  • 複数入口: Python API、CLI、Web Demo、Playground、Docs が揃っており試しやすいです。

リポジトリの構成

コア実装と周辺デモが分離されており、研究コードをそのまま使うだけでなく、プロダクト導入の足場もあります。

  • src/voxcpm/: モデル本体の Python パッケージ。
  • conf/: 設定ファイル群。
  • examples/、scripts/: 利用例と補助スクリプト。
  • app.py、lora_ft_webui.py: デモや fine-tuning 入口。

セットアップ・導入のイメージ

最短導入は Python パッケージ voxcpm のインストールです。README では Python 3.10 以上、PyTorch 2.5 以上、CUDA 12.0 以上が前提になっています。

  1. pip install voxcpm を実行します。
  2. VoxCPM.from_pretrained("openbmb/VoxCPM2") でモデルを読み込みます。
  3. generate() に通常テキスト、声質記述、参照音声を与えて TTS、Voice Design、Voice Cloning を切り替えます。

ターゲット層 / 注目ポイント

多言語ナレーション生成、ゲームや映像向け音声制作、対話エージェントの音声化など、質感重視の音声生成案件に向いています。 特に voice design と controllable cloning が同じ生成 API に乗っているため、単純な読み上げツールではなく、音声制作エンジンとして見られている点がトレンド入りの理由です。

制約・リスク

高品質推論には GPU 条件が重く、ローカル導入の敷居は低くありません。さらにボイスクローニング機能は利用規約や権利処理の設計が不可欠で、プロダクト実装では技術より運用ガードが重要になります。

まとめ

VoxCPM は、単なる OSS TTS ではなく、声質生成と制御を一体化した音声モデル基盤として完成度が高いです。 研究成果の公開にとどまらず、今の生成音声 UI や API の実装候補として見る価値があります。
