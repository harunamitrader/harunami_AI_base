GitHub Watcher

Gitleaks

コード内に潜むパスワードやAPIキーを自動検出。Git履歴を遡り、機密情報の流出を未然に防ぐ強力なセキュリティツール。

これは何か

リポジトリやファイルの中にハードコードされた機密情報（Secrets）を検出するための、デファクトスタンダード的なセキュリティツールです。パスワード、APIキー、AWSアクセスキー、プライベートトークンなどが誤ってコミットされていないかを、超高速にスキャンします。

何ができるか

単に現在のファイルをチェックするだけでなく、Gitの全コミット履歴を遡ってパッチ単位で解析します。正規表現とエントロピー解析を組み合わせた高精度なエンジンにより、数千コミットを超えるリポジトリでも数秒でスキャンを完了し、漏洩の証拠（Fingerprint）を特定します。

目立つポイント

圧倒的な「使いやすさ」と「エコシステム」が特徴です。CLI単体での利用はもちろん、pre-commit フックとしてコミット前に自動チェックしたり、公式の GitHub Action を利用してプルリクエスト時に自動で機密情報漏洩をブロックしたりと、開発プロセスのあらゆる段階に導入可能です。

セットアップや使い方の流れ

Go製のシングルバイナリで、Homebrew (brew install gitleaks) や Docker を通じて簡単に導入できます。gitleaks git -v を実行すればカレントリポジトリの履歴を含めたフルスキャンが開始されます。また、.gitleaks.toml をリポジトリに配置することで、独自のルール追加や特定パスの除外も自由自在です。

どんな人向けか

公開リポジトリを運用している個人開発者はもちろん、DevSecOpsを推進し、機密情報の流出による致命的な事故を未然に防ぎたい企業の開発チームに必須のツールです。特に、AWSやGitHubトークンを多用するクラウドネイティブな開発環境において威力を発揮します。

注意点

万が一機密情報をコミットしてしまった場合、Gitleaksで場所を特定し、ファイルを修正しただけでは不十分です。Git履歴に残っている情報は依然として危険であるため、キーの無効化（ローテーション）と、必要に応じて履歴の書き換え（git filter-repo 等）を行う必要があります。

まとめ

「機密情報のコミット」というミスは、どれだけ注意していても起こり得るものです。Gitleaksをパイプラインに組み込むことは、そうしたヒューマンエラーが組織的な事故に発展するのを防ぐ、最も安価で効果的な保険と言えるでしょう。
