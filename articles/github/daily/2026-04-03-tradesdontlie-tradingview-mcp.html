これは何か

tradingview-mcp は、TradingView Desktop を Claude Code などの AI アシスタントへつなぐ MCP ブリッジです。README では Chrome DevTools Protocol 経由でローカルの TradingView アプリに接続し、AI-assisted chart analysis や Pine Script development を行うための tool と説明されています。

何ができるか

シンボル変更、時間足変更、ズーム、インジケーター追加、描画、アラート管理、リプレイ練習、スクリーンショット取得、マルチペイン構成、Pine Script の注入とデバッグまで扱えます。さらに MCP ツール群と同じ機能が tv CLI でも使えるので、AI 連携だけでなくローカル自動化にも広げやすい構成です。

目立つポイント

特徴は「外部 API へつなぐ」のでなく、あくまで自分のローカル TradingView Desktop を CDP で操作する点です。README でも、TradingView のサーバーへ接続せず、データを外部送信せず、課金制限を回避しないことを強く明記しています。つまりこれは取引 bot というより、トレーディング画面を AI に読み書きさせる interface layer です。

セットアップや使い方の流れ

基本の流れは、リポジトリを clone して npm install、TradingView Desktop を --remote-debugging-port=9222 付きで起動し、Claude Code の MCP 設定へ src/server.js を追加する形です。その後に tv_health_check で接続確認し、必要な chart 操作や Pine Script 作業を AI へ任せます。

どんな人向けか

TradingView を日常的に使っていて、Pine Script の試作、チャート操作、検証補助を AI に手伝わせたい人に向いています。特にチャート画面を AI に「見せたい」けれど、クラウドへデータを渡したくない人にはかなり分かりやすい選択肢です。

注意点

README では、TradingView の有効なサブスクリプションと Desktop アプリが必要で、内部の undocumented Electron API に依存するため、TradingView 更新で壊れる可能性があると明記されています。また README 自身もこれは real trades を実行しないと書いているので、売買執行ではなく分析補助用として見るべきです。

まとめ

tradingview-mcp は、TradingView Desktop を Claude Code などに接続して、ローカルチャートを AI で扱えるようにする実験色の強い実務ツールです。トレード執行ではなく、チャート分析と Pine Script 開発の補助レイヤーとして見ると価値が分かりやすいリポジトリです。
