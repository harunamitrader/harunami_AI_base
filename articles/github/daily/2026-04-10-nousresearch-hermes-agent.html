プロジェクトの概要

hermes-agent は、Nous Research が開発する常駐型の AI エージェント基盤です。 CLI だけでなく Telegram、Discord、Slack、WhatsApp、Signal などのメッセージング経路を単一ゲートウェイで扱いながら、作業経験からスキルを作り、記憶を蓄積し、次回以降に自己改善する閉ループを中核に置いています。

機能・できること

  • 自己改善ループ: 経験からスキルを作成し、利用中に改善し、過去会話を検索して再利用します。
  • マルチチャネル運用: CLI と複数メッセージングプラットフォームをまたいで同一エージェントを維持できます。
  • サブエージェント並列化: 独立したサブエージェントを起動し、並列作業やパイプライン化が可能です。
  • 広い実行基盤: ローカル、Docker、SSH、Daytona、Singularity、Modal など複数バックエンドに対応します。

リポジトリの構成

エージェント本体だけでなく、CLI、ゲートウェイ、スキル、プラグイン、研究用途まで含めた大きめの構成です。

  • agent/、hermes_cli/: コアエージェントと CLI まわり。
  • gateway/: Telegram や Discord などへ接続するメッセージング層。
  • skills/、plugins/、tools/: 拡張と能力追加の仕組み。
  • cron/、tests/、docs/: 自動化、検証、ドキュメント。

セットアップ・導入のイメージ

Quick Install はシェルスクリプト 1 本で、Linux、macOS、WSL2 を対象にしています。Windows ネイティブは非対応で、README でも WSL2 利用が明記されています。

  1. curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash で導入します。
  2. hermes model で利用する LLM を選び、hermes tools や hermes setup で環境を整えます。
  3. 必要に応じて hermes gateway を起動し、CLI 外のチャネルとも連携します。

ターゲット層 / 注目ポイント

「ローカルの補助ツール」ではなく、「常にどこかで動き続ける作業エージェント」を作りたい人に向いています。 モデルロックインを避けつつ、記憶、スキル、並列化、スケジューリングまで一式揃っているため、個人用 AI 秘書から運用エージェントまで用途が広いです。

制約・リスク

機能が多いため、初期設定や権限制御の設計を雑にすると扱いづらくなります。外部チャネルや長期記憶を有効にする場合は、秘密情報、作業ディレクトリ、許可コマンドの境界設計が実質的な本番課題です。

まとめ

hermes-agent は、対話 UI 付きの CLI を超えて、学習し続ける作業エージェントの実装見本として完成度が高いです。 2026 年時点のオープンな agent 基盤の中でも、常駐運用と自己改善をここまで一体化している例は少なく、1 位になる理由は明確です。
