これは何か

hermes-agent-self-evolution は、Hermes Agent の skill、tool description、system prompt、将来的にはコード自体までを自動で改善していくための実験基盤です。README では DSPy と GEPA を使った evolutionary self-improvement と説明されており、学習用 GPU なしで API 呼び出しベースに最適化を回す設計です。

何ができるか

現状の中心は Phase 1 の skill file 最適化で、既存の SKILL.md を読み込み、評価データを作り、候補を複数生成し、実行結果から良い変種を選びます。評価データは synthetic 生成だけでなく、Claude Code や Copilot、Hermes の session history を使う流れも想定されています。

目立つポイント

単なる AB テストではなく、execution trace を読んで「なぜ失敗したか」まで踏まえて改善案を出すのが特徴です。さらに guardrail として test 全通過、サイズ制限、キャッシュ互換、意味的な逸脱防止、人間レビュー必須が置かれており、進化させつつ暴走を抑える構成になっています。

セットアップや使い方の流れ

Quick Start は短く、リポジトリを clone して pip install -e ".[dev]" を実行し、環境変数 HERMES_AGENT_REPO に Hermes Agent 本体のパスを指定します。その上で python -m evolution.skills.evolve_skill --skill github-code-review --iterations 10 のように実行し、skill 単位で進化を回す流れです。

どんな人向けか

エージェントの prompt や skill を人手で少しずつ調整する運用に限界を感じている人に向いています。特に「評価データはある」「どこが弱いかは何となく分かるが、改善の当たりを引くのが大変」というチームには相性が良さそうです。

注意点

README 上でも実装済みなのは skill 最適化が中心で、tool descriptions、system prompt、code evolution、continuous loop はまだ planned が多い段階です。また対象は Hermes Agent 前提なので、他の agent へそのまま流用するには評価設計や制約を組み替える必要があります。

まとめ

hermes-agent-self-evolution は、AI エージェントの挙動改善を「勘で prompt を直す作業」から、評価と進化ループのある反復プロセスへ寄せようとするリポジトリです。skill 改善を継続運用したい人にとって、かなり参考になる設計です。
