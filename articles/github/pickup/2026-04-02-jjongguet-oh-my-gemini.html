これは何か

oh-my-gemini は、Gemini CLI 向けに CLI 拡張と運用コマンドをまとめたプロジェクトです。README では「Gemini CLI のための multi-agent orchestration」を掲げており、通常の対話実行に加えて複数ワーカーの並列作業や状態監視を扱えるようにしています。

何ができるか

グローバルに oh-my-gemini-sisyphus を入れて omg setup --scope project を実行すると、Gemini CLI に /omg:plan、/omg:review、/omg:verify などのコマンド群を追加できます。加えて omg team run で複数ワーカー実行、omg team status や omg hud --watch で進捗確認、omg doctor で前提チェックまで一続きで回せます。

目立つポイント

特徴は、単なるプロンプト集ではなくチーム運用の状態を .omg/state/ に保持し、resume や shutdown を前提にしている点です。tmux を既定バックエンドにしつつ、役割タグ付きの subagents 実行も選べるため、Gemini CLI を 1 セッションずつ手で回すより再現性のある進行管理を作りやすい構成です。

セットアップや使い方の流れ

基本導入は Node.js 20+、Gemini CLI、tmux を用意して npm からインストールし、omg setup --scope project で拡張を登録、その後に Gemini CLI を再起動します。日常利用は omg で対話起動、並列タスクは omg team run --task \"...\" --workers 2、確認は omg team status や omg hud --watch という流れが分かりやすいです。

どんな人向けか

Gemini CLI を単発の会話だけでなく、レビュー、検証、長めの実装タスクへ広げたい人に向いています。特に tmux を使った分業や、実行途中の再開、チーム状態の可視化が欲しい開発者には相性がよさそうです。

注意点

README 上の必須要件に Node.js 20+、Gemini CLI、tmux が含まれているため、Windows では WSL2 前提の tmux セットアップを考えた方が安全です。また omg verify は開発リポジトリ内向けの検証セットという記述があるので、外部プロジェクトでそのまま全て使える前提では見ない方がよいです。

まとめ

oh-my-gemini は、Gemini CLI を「1 人で対話する道具」から「並列実行と検証を含む作業基盤」へ広げるための実務寄りラッパーです。Gemini を使った開発フローを標準化したい人には、導入コストに対して得られる運用整理の効果が大きいリポジトリです。
