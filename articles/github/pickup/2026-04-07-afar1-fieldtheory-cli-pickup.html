これは何か

fieldtheory-cli は、X (旧Twitter) のブックマークをローカル環境に同期・保存・管理するためのオープンソースのコマンドラインツールです。 公式 API の制限を回避しながら、Chrome のセッションを利用して情報を取得し、自分だけの検索可能なナレッジベースを構築できます。

何ができるか

  • ブックマーク同期 (ft sync): 有料 API 不要で Chrome からセッションを抽出して同期。
  • 高速な全文検索 (ft search): BM25 アルゴリズムを用いたランキング検索。
  • ビジュアライゼーション (ft viz): ブックマークの分布などをダッシュボードで視覚化。
  • AI 分類 (ft sync --classify): LLM を活用したブックマークの自動カテゴリー分け。

目立つポイント

最大の強みは「ローカルファースト」かつ「API 不要」である点です。データはすべて ~/.ft-bookmarks/ に保存され、プライバシーが確保された状態でオフライン検索が可能です。 また、AI エージェントが過去のブックマークを「記憶」として参照するための基盤としても非常に有用です。

セットアップや使い方の流れ

  1. Node.js 20 以上と X ログイン済みの Google Chrome を用意します。
  2. npm install -g fieldtheory でインストール。
  3. ft sync を実行してブックマークをローカルに同期。
  4. ft search "キーワード" で目的の情報を瞬時に見つけ出せます。

どんな人向けか

  • X で大量の情報を保存しているが、公式の検索機能やスクロールに限界を感じている人。
  • AI エージェント (Claude Code, Cursor 等) に過去のブックマーク情報を引用させたい開発者。
  • 自分だけの技術ナレッシベースをローカルに構築したい人。

注意点

現在は Mac 環境に最適化されているため、Windows や Linux 環境ではパスや依存関係の調整が必要になる場合があります。また、公式 API を介さない手法であるため、基盤となる X の UI 仕様変更による影響を受ける可能性があります。

まとめ

fieldtheory-cli は、散らばりがちな X のブックマークを、整理・検索可能な「資産」へと変えてくれる強力なツールです。特に AI を活用した開発効率化を進めたいユーザーにとって、過去に保存した断片的な情報を即座にコンテキストとして活用できるメリットは計り知れません。
