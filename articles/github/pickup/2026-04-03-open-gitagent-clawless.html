clawless

Pickup #051

Claw 系 AI エージェントを、サーバーなしでブラウザ内だけで動かせる WebContainers ベースの実行基盤。

記事作成日 2026-04-03
種別 Pickup
対象 repo open-gitagent/clawless

これは何か

clawless は、Claw 系 AI エージェントをサーバーなしでブラウザ内実行できる runtime です。README では WebContainers による serverless browser-based runtime と説明しており、Node.js 実行環境、仮想ファイルシステム、ターミナル、エディタをすべてブラウザ内に閉じ込めています。

何ができるか

エージェントの起動、コード生成と実行、ファイル読み書き、ライブプレビュー、GitHub clone / push、監査ログ収集、YAML ベースのポリシー制御までを扱えます。README の例では pptxgenjs をブラウザ内でインストールして PowerPoint を生成するデモまで載っており、単なるチャット UI よりかなり実行環境寄りです。

目立つポイント

特徴は「サーバーレス」と言っても軽い sandbox ではなく、監査とガードレールをかなり前面に出している点です。プロセス起動、ファイル I/O、ネットワーク要求、git 操作まで監査対象になり、ポリシーは YAML でファイルアクセス、実行プロセス、ポート、ファイルサイズ、turn 数、timeout まで制御できます。ブラウザだけでエージェント環境を配りたい用途にはかなり噛み合っています。

セットアップや使い方の流れ

README の quick start では、まずローカルで clone して npm install、npm run dev で起動する流れです。組み込み利用では clawcontainer を依存追加し、ClawContainer を DOM 要素へマウントして、template と API key を渡して起動します。つまりアプリとして触るだけでなく、SDK として自分のサービスへ埋め込む前提もあります。

どんな人向けか

AI エージェントをユーザー環境へ安全に配りたい人、サーバーコストを持たずにブラウザ完結の agent runtime を試したい人、実行監査やポリシー制御を重視したい人に向いています。特に「ローカルマシンへ直接触らせたくないが、コード実行は必要」という場面では相性がよさそうです。

注意点

ブラウザ内 Node.js runtime と WebContainers 前提なので、重い依存や長時間ジョブでは通常のサーバー実行より制約を受ける可能性があります。また完全に host system から隔離される設計上、ローカル資産へ直接触るワークフローはそのまま移植できません。用途は「安全な仮想実行環境」に寄せて考える方が自然です。

まとめ

clawless は、Claw 系エージェントをブラウザだけで動かしたい人向けの、実行基盤寄りなリポジトリです。WebContainers に監査、ポリシー、SDK を重ねており、配布可能な agent sandbox を作りたい人にはかなり参考になります。
