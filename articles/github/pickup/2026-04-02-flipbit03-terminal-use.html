terminal-use

Pickup #049

AI エージェントが GUI なしで terminal UI を起動し、画面を見てキー入力できる仮想ターミナル。

記事作成日 2026-04-02
種別 Pickup
対象 repo flipbit03/terminal-use

これは何か

terminal-use は、AI エージェント向けのヘッドレス仮想ターミナル CLI です。README では「interactive terminal apps を spawn し、screen を読み、keystrokes を送れる」と説明されており、GUI や X server なしで ncurses 系アプリを扱えるようにします。

何ができるか

htop、vim、mc、dialog ベースのインストーラのような TUI を仮想端末で起動し、描画された画面を取得して、AI エージェント側からキー入力を送れます。加えて tu monitor で別端末からセッション監視もでき、複数セッションの切り替えやリサイズにも対応します。

目立つポイント

特徴は、標準入出力の単純な pipe では扱えない terminal UI を、PTY と vt100 エミュレータを介して stateless な CLI として公開している点です。構成は CLI から Unix socket 越しに daemon へ接続し、daemon が PTY と terminal emulator を管理する形で、初回利用時の自動起動と 8 時間無操作後の自動終了も整理されています。

セットアップや使い方の流れ

導入は prebuilt binary を install script で入れるか、Rust 環境から cargo install terminal-use で入れます。利用時はまず tu usage でコマンド体系を確認し、その後に対象アプリを仮想端末で起動して、必要なら別ターミナルで tu monitor を開いて挙動を追う流れが分かりやすいです。

どんな人向けか

AI エージェントにブラウザだけでなく terminal UI も触らせたい人に向いています。特にサーバー上の TUI 操作、インストーラの対話画面、ゲームや運用ツールの自動操作のように、GUI なし環境で画面状態を見ながら進めたい用途と相性がよさそうです。

注意点

README によると prebuilt binary は Linux と macOS 向けで、実装も Unix socket 前提なので、Windows ネイティブ利用はそのままでは難しい可能性があります。また terminal UI の操作はキー送信単位の制御になるため、高速な画面変化や複雑な状態管理ではエージェント側の観測設計が重要です。

まとめ

terminal-use は、AI エージェントに「本物の terminal を触らせる」ための薄くて実用的な基盤です。TUI がボトルネックになっていた自動化や検証フローでは、かなり面白い選択肢になりそうです。
